## 최종 목표 형태
`SHealth::calculateBmi`는 최종적으로 이런 수준의 오케스트레이터가 되면 좋습니다.

```cpp
int SHealth::calculateBmi(const std::string& filename) {
    resetState();

    auto loadedRecords = loadRecords(filename);
    if (!loadedRecords) {
        return 0;
    }

    records = std::move(*loadedRecords);
    imputeMissingWeights(records);
    calculateBmiValues(records);
    ratiosByAgeClass = aggregateRatios(records);

    return static_cast<int>(records.size());
}
```

가장 중요한 커밋 경계는 `상수화`, `파일 파싱 분리`, `vector 기반 레코드 도입`, `BMI 정책 테이블화`, `ratio map 저장 구조 변경`입니다. 이 순서로 가면 각 커밋마다 `cmake --build build`와 `ctest --test-dir build`로 회귀 여부를 확인할 수 있습니다.

- 이 테스트가 통과하면 BMI 정책 다형성이 실제로 동작하는 것이다.

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

1. `BmiCategory` 타입 분리
2. `BmiClassifier` 인터페이스 추가

현재 코드 규모에서는 전략 패턴을 적용하되, BMI type code 변환까지 classifier에 넣지는 않는 편이 좋습니다. classifier는 “BMI 수치 -> BMI 카테고리”만 책임지고, `100/200/300/400` 호환 코드는 `SHealth`의 조회 API 경계에 두는 구조가 가장 깔끔합니다.
