요청하신 Golden Master 회귀 테스트를 현재 C++/GTest/CMake 구조에 맞춰 구현하겠습니다. 먼저 참조 파일들과 빌드 설정을 읽어서 출력 흐름과 기존 테스트 패턴을 확인하겠습니다.

`TexttestFixture.cpp`는 지정된 경로 그대로는 없어서 위치를 더 찾아보겠습니다. 동시에 기존 `SHealthBMITest`와 루트 `CMakeLists.txt`, 입력 데이터 파일 구성을 확인해 테스트를 자연스럽게 붙이겠습니다.

현재 저장소에는 `TexttestFixture.cpp`가 없어서, 그 이름으로 Golden Master 전용 GTest 파일을 새로 추가하는 방향이 가장 자연스럽습니다. 기준 출력은 기존 `SHealthBMI.cpp`의 콘솔 출력 포맷과 `shealth.dat`를 고정 fixture로 삼아 `tests/golden/...expected.txt`에 보관하겠습니다.

수정 범위는 작게 잡겠습니다. `TexttestFixture.cpp`에 출력 렌더링과 파일 비교를 두고, Golden 파일은 소스 트리의 `tests/golden`에 고정하며, CMake는 별도 테스트 타깃과 `ctest` 자동 발견만 추가하겠습니다.

구현 파일과 Golden 기준 파일, CI 워크플로를 추가했습니다. 이제 CMake 재구성 후 새 테스트 타깃이 `ctest`에 등록되고 통과하는지 확인하겠습니다.

이 환경의 PowerShell이 `&&` 체인을 해석하지 못해 명령 자체가 시작 전에 실패했습니다. 같은 검증을 PowerShell 호환 방식으로 다시 실행하겠습니다.

검토하면서 Golden Master의 핵심인 “실제 프로그램 출력”을 더 잘 잠그려면 테스트가 `SHealthBMI` 실행 파일의 stdout을 비교해야 한다고 판단했습니다. 테스트 내부에서 포맷을 재구성하지 않고 실제 실행 결과를 캡처하도록 `TexttestFixture.cpp`를 조정하겠습니다.

이제 Golden 테스트가 실제 `SHealthBMI` 실행 파일을 소스 루트에서 실행해 `shealth.dat`를 읽고 stdout을 비교합니다. 수정된 CMake 정의까지 포함해 다시 빌드와 전체 테스트를 돌리겠습니다.

구현 완료했습니다.

추가/수정한 항목은 다음입니다.

- `src/test/cpp/TexttestFixture.cpp`: 실제 `SHealthBMI` 실행 파일을 실행해서 stdout을 캡처하고 Golden 파일과 비교하는 Google Test
- `tests/golden/shealth_bmi.expected.txt`: 현재 `shealth.dat` 기준 expected 출력 보관
- `CMakeLists.txt`: `SHealthGoldenMasterTest` 타깃 추가 및 `ctest` 등록
- `.github/workflows/ci.yml`: push/PR에서 CMake build + `ctest` 자동 실행

Golden Master 전략은 `shealth.dat`와 콘솔 출력 포맷을 회귀 기준으로 고정하고, 의도된 출력 변경이 있을 때만 `tests/golden/shealth_bmi.expected.txt`를 갱신하는 방식입니다. 갱신은 repo root에서 예를 들어:

```powershell
cmd /c "build\Debug\SHealthBMI.exe > tests\golden\shealth_bmi.expected.txt"
```

실행은 다음처럼 하면 됩니다.

```powershell
cmake -S . -B build
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

검증 결과: 전체 `21/21` 테스트 통과했습니다. `build/`는 로컬 CMake 생성물로 계속 untracked에 남아 있습니다.

요구사항을 `ApprovalTests::Approvals::verify` 기반으로 바꾸는 작업으로 이해했습니다. 먼저 ApprovalTests C++의 현재 CMake/Google Test 연동 방식을 확인하고, 기존 `TexttestFixture.cpp`와 CMake를 그 방식에 맞춰 조정하겠습니다.

Context7 문서 조회는 quota 초과로 실패했습니다. 공개 문서/저장소 정보와 현재 프로젝트 구조를 바탕으로 `ApprovalTests.cpp` FetchContent 연동 방식으로 진행하겠습니다.

ApprovalTests.cpp는 `gtest_main`만 쓰면 승인 파일 이름을 안정적으로 얻지 못한다는 문서 제약이 있습니다. 그래서 Golden Master 전용 테스트 실행 파일에는 ApprovalTests가 제공하는 Google Test main을 두고, 테스트 본문에서는 `Approvals::verify(actualOutput)`만 호출하도록 바꾸겠습니다.

편집은 세 부분입니다. `ApprovalTestsMain.cpp`를 추가해 승인 파일 디렉터리를 고정하고, `TexttestFixture.cpp`는 실제 실행 출력 문자열을 `Approvals::verify()`에 넘기도록 단순화하며, CMake는 ApprovalTests.cpp를 `FetchContent`로 받아 전용 테스트 타깃에 링크하겠습니다.

기존 수동 비교 Golden 파일은 제거하고, ApprovalTests의 `.approved.txt` 규칙으로 기준 파일을 옮겼습니다. 이제 의존성 다운로드, 빌드, `ctest` 등록/실행까지 확인하겠습니다.

빌드와 테스트 등록은 성공했지만 Approval 비교에서 줄 끝이나 파일 끝 개행 차이로 보이는 mismatch가 났습니다. ApprovalTests가 만든 `.received.txt`를 확인해서 승인 파일과 정확히 맞추겠습니다.

차이는 출력 내용이 아니라 ApprovalTests가 문자열 저장 시 붙이는 마지막 줄 처리와 실제 stdout의 마지막 개행이 겹친 문제입니다. 비교 대상 문자열의 trailing newline만 정규화해서 의미 있는 출력 라인만 승인 대상으로 고정하겠습니다.

`ApprovalTests::Approvals::verify()` 기반으로 전환했습니다.

변경 사항:
- `src/test/cpp/ApprovalTestsMain.cpp`: ApprovalTests Google Test main 추가, 승인 파일을 `approval_tests/` 하위에 저장하도록 설정
- `src/test/cpp/TexttestFixture.cpp`: 실제 `SHealthBMI` stdout을 캡처한 뒤 `ApprovalTests::Approvals::verify(actual)`로 검증
- `src/test/cpp/approval_tests/TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.approved.txt`: Golden Master 기준 출력
- `CMakeLists.txt`: `ApprovalTests.cpp v.10.13.0` FetchContent 추가, `SHealthGoldenMasterTest`를 `ctest`에 등록
- `.github/workflows/ci.yml`: push/PR에서 CMake build + `ctest` 실행

승인 파일 갱신 전략은 ApprovalTests 표준 흐름입니다. 출력 변경이 의도된 경우 테스트 실패 시 생성되는 `.received.txt`를 검토한 뒤 `.approved.txt`로 교체합니다.

```powershell
copy src\test\cpp\approval_tests\TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.received.txt src\test\cpp\approval_tests\TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.approved.txt
```

실행 방법:

```powershell
cmake -S . -B build
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

검증 완료: 전체 `21/21` 테스트 통과했습니다. CMake는 `FetchContent_MakeAvailable()` 사용에 맞춰 최소 버전을 `3.14`로 올렸습니다.
