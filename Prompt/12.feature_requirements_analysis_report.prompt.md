요청하신 `README.md`의 “4. 기능개선” 내용을 C++17 구현과 QA 관점에서 재정리해 `docs/feature_requirements_analysis.md`로 작성하겠습니다. 먼저 현재 README의 해당 섹션과 기존 문서 스타일을 확인하겠습니다.

기존 문서가 이미 QA 관점의 요구사항/테스트 계획을 갖고 있어서, 새 문서는 README의 4번 개선 항목을 구현 단위, 공개 API 후보, 테스트 시나리오 번호까지 연결하는 형태로 맞추겠습니다. 구현 세부와 현재 클래스 구조를 확인한 뒤 파일을 작성하겠습니다.

현재 구현은 이미 `SHealth` 안에서 로딩, 파싱, 보정, BMI 계산, 집계가 함수 단위로 나뉘어 있고 `BmiClassifier`가 분리되어 있습니다. 새 문서에는 이 상태를 기준으로 추가 분리 후보와 기능별 API/데이터 구조/테스트 번호를 정리하겠습니다.

이제 `docs/feature_requirements_analysis.md`를 새로 추가하겠습니다. README의 4번 항목 순서를 유지하되, 각 개선 항목을 구현 책임, 제안 인터페이스, 예외/경계 조건, Google Test 시나리오 번호로 추적 가능하게 작성합니다.

작성 완료했습니다.

`docs/feature_requirements_analysis.md`에 `README.md`의 “4. 기능 개선” 항목을 C++17 구현 관점으로 재정리했습니다. 포함 내용은 다음과 같습니다.

- SRP 기반 책임 분리 리팩토링 방향
- 연령대별 BMI 분포 비율 계산 요구사항
- `height == 0` 평균 키 보정 로직
- BMI 정상 범위 사용자 목록 조회 요구사항
- 전체 사용자 대비 BMI 범주 비율 계산 요구사항
- Google Test 기준 테스트 시나리오 `TC-FR-001` ~ `TC-FR-020`

`ReadLints` 확인 결과 문서 파일에 린트 오류는 없습니다.
