요청하신 기능별 테스트를 기존 Google Test 스타일에 맞춰 추가하겠습니다. 먼저 관련 헤더/구현/기존 테스트를 읽어서 현재 API와 테스트 패턴을 확인하겠습니다.

현재 구현은 연령대별 비율과 체중 0 보정만 공개되어 있고, 요구사항의 `height == 0` 보정, 정상 BMI 사용자 조회, 전체 범주 비율 API는 아직 없습니다. 테스트를 Green으로 만들려면 테스트 추가와 함께 필요한 최소 공개 API/구현을 같이 보강하겠습니다.

기존 테스트는 이미 BMI 분류와 연령대 비율을 많이 다루고 있지만, 새 요구사항 4개 중 3개는 구현 표면이 없습니다. 공개 API는 `getNormalBmiUsers()`와 `getOverallBmiRatio(type)` 정도로 작게 추가하고, 기존 `getBmiRatio(ageClass,type)` 동작은 유지하는 방향으로 진행하겠습니다.

이제 `SHealth`에 테스트 가능한 최소 API와 내부 상태를 추가합니다. CSV의 ID를 보존할 수 있도록 레코드 모델도 공개 조회용 구조체로 정리하겠습니다.

다음은 구현부입니다. `height == 0`은 같은 연령대의 유효 키 평균으로 보정하고, 보정 불가 또는 BMI 0인 레코드는 통계 분모에서 제외되도록 기존 패턴에 맞춰 넣겠습니다.

테스트는 기존 픽스처와 임시 CSV 작성 헬퍼를 그대로 사용해 추가합니다. 부동소수 오차 비교 없이 `EXPECT_EQ/ASSERT_EQ`만 쓰기 위해 비율은 25/50/100처럼 정확히 검증 가능한 데이터로 구성하겠습니다.

코드와 테스트 추가가 끝났습니다. 이제 기존 빌드 디렉터리에서 `cmake --build build`와 `ctest`를 순서대로 실행해 컴파일 오류와 테스트 실패를 잡겠습니다.

빌드와 전체 CTest가 Green입니다. 마지막으로 수정한 C++ 파일에 IDE 진단이 있는지 확인하겠습니다.

요청하신 `ctest` 실행 형태도 빌드 디렉터리 기준으로 한 번 더 확인하겠습니다. Green 상태가 유지되는지만 빠르게 검증합니다.

완료했습니다.

`SHealth`에 필요한 기능 구현을 보강하고, `SHealthBMITest.cpp`에 요청하신 4개 기능별 5개씩 총 20개 테스트를 추가했습니다. 모든 테스트는 Given-When-Then 주석 구조와 `EXPECT_EQ`/`ASSERT_EQ` 검증을 사용했습니다.

변경 요약:
- `height == 0`을 같은 연령대 평균 키로 보정
- CSV 사용자 ID 보존
- `getNormalBmiUsers()` 추가
- `getOverallBmiRatio(type)` 추가
- 연령/키/체중 경계값 및 제외 케이스 테스트 추가

검증 완료:
- `cmake --build build` 성공
- `ctest` 성공
- 전체 `41/41` 테스트 Green
- 수정 파일 linter 오류 없음
