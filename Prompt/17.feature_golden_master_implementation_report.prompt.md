요청하신 Golden Master 회귀 테스트를 현재 테스트 구조에 맞춰 구현하겠습니다. 먼저 기존 ApprovalTests 진입점, fixture, CMake 구성을 읽어서 이미 쓰는 패턴을 확인하겠습니다.

이미 ApprovalTests 기반 테스트가 하나 들어가 있지만, 현재는 실행 파일 출력만 검증하고 승인 파일 보관/비교 흐름이 다소 약합니다. 이제 루트 CMake와 CI 워크플로를 확인해 ctest에 자연스럽게 붙이겠습니다.

이제 `TexttestFixture.cpp`에 신규 기능을 한 번에 보여주는 deterministic snapshot을 추가하고, CMake에는 Golden Master 테스트용 expected 디렉터리와 ctest 라벨을 연결하겠습니다. CI는 기존 `ctest` 실행을 유지하되 실패 시 Approval received 파일을 artifact로 남기도록 보강하겠습니다.

수정은 끝났고, 이제 CMake 재구성과 테스트를 돌려 실제 ctest 등록 및 Golden Master 비교가 통과하는지 확인하겠습니다.

현재 PowerShell이 `&&` 체인을 지원하지 않아 명령 파싱에서 멈췄습니다. 같은 검증을 PowerShell 호환 형식으로 다시 실행하겠습니다.

불필요한 include 하나를 정리했습니다. 같은 테스트 명령으로 다시 확인해서 최종 상태를 고정하겠습니다.

Expected 파일 갱신 전략을 더 분명하게 하기 위해, 파일 비교가 실패하면 실제 출력도 `.received.txt`로 남기도록 보강하겠습니다. 그러면 CI에서도 실패 산출물을 내려받아 승인 여부를 검토할 수 있습니다.

구현 완료했습니다.

`src/test/cpp/TexttestFixture.cpp`에 신규 기능용 Golden Master 테스트 `SHealthFeatureSnapshotMatchesExpectedFile`를 추가했습니다. 이 테스트는 임시 CSV 입력을 만들고 다음 기능 출력을 하나의 snapshot으로 비교합니다: 키 `0` 평균 보정, 연령대별 BMI 비율, 전체 BMI 비율, 정상 BMI 사용자 목록.

Expected 출력 파일은 여기에 보관됩니다:

`src/test/cpp/golden_master/SHealthFeatureSnapshot.expected.txt`

비교 실패 시 실제 출력은 다음 파일로 남도록 했습니다:

`src/test/cpp/golden_master/SHealthFeatureSnapshot.received.txt`

CMake에는 Golden Master expected 디렉터리 define, `SHealthBMI` 실행 파일 dependency, ctest 라벨(`unit`, `golden-master`)을 추가했습니다. CI는 기존처럼 `ctest` 전체를 자동 실행하고, 실패 시 Approval/Golden Master received 파일을 artifact로 업로드하도록 보강했습니다.

실행 방법:

```powershell
cmake -S . -B build
cmake --build build --config Release
ctest --test-dir build -C Release --output-on-failure
```

검증 결과: 전체 `42/42` 테스트 통과, linter 오류 없음.

참고로 검증 과정에서 `build/` 디렉터리가 생성되었습니다. `build-coverage/`는 작업 시작 전부터 untracked 상태였습니다.
