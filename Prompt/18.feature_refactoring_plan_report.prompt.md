**Cursor**

요청하신 범위는 `SHealth.cpp` 중심의 리팩토링 계획이므로, 먼저 지정하신 헤더/구현/문서/CLI 파일을 읽어 현재 구조와 테스트 제약을 확인하겠습니다.

현재 구현은 이미 일부 상수화와 `BmiClassifier` 전략 분리가 되어 있지만, 평균 보정/비율 집계/타입 매핑 쪽에 중복과 정책 혼재가 남아 있습니다. 테스트 Green 유지 조건에 맞춰 작은 커밋 단위로 계획을 정리하겠습니다.

현재 `SHealth.cpp`는 기능은 꽤 잘 테스트되어 있고, 리팩토링은 “동작 고정 → 중복 제거 → 정책 분리 → 파일 분리” 순서가 안전합니다. 각 단계는 반드시 Green 확인 후 다음 단계로 넘어가는 방식이 좋습니다.

검증 명령은 매 커밋마다 동일하게 잡겠습니다.

```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

## 단계별 리팩토링 계획

### 0. Baseline Green 고정
체크리스트:
- 현재 테스트가 Green인지 먼저 확인
- 실패가 있으면 리팩토링 시작 전 원인 분리
- `SHealth`의 현재 공개 동작을 변경 금지:
  - 잘못된 파일은 `0` 반환
  - 잘못된 BMI type은 `0.0`
  - 연령대별 통계는 `20~79`만 포함
  - 전체 통계는 유효 BMI면 연령대 밖도 포함
  - unresolved `height == 0`은 통계 제외

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

커밋 예시:
- `test: confirm current BMI behavior before refactoring`

### 1. 매직 넘버와 BMI 타입 매핑을 테이블 기반으로 정리
대상:
- `categoryFromType()`
- `RatioSet`의 `std::array<double, 4>`
- `std::array<int, 4> counts`
- `100/200/300/400`
- `100.0` percentage scale

체크리스트:
- `BmiTypeMappings`를 `constexpr std::array<std::pair<int, BmiCategory>, 4>`로 도입
- `switch`를 테이블 조회로 변경
- `CategoryCount`, `PercentageScale` 상수 도입
- `static_cast<std::size_t>(category)` 사용 위치를 `toCategoryIndex()` 같은 함수로 모음
- public API `getBmiRatio(int ageClass, int type)`와 `getOverallBmiRatio(int type)`는 그대로 유지

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

커밋 예시:
- `refactor: replace BMI type switch with category table`

### 2. 연령대 정책을 `AgeClassPolicy`로 모으기
대상:
- `MinAgeClass`
- `MaxAgeClass`
- `AgeClassStep`
- `AgeClasses`
- `toAgeClass()`
- `aggregateRatios()`

체크리스트:
- 연령대 배열과 범위 판단을 한 정책으로 묶음
- `AgeClasses`와 `toAgeClass()`가 서로 다른 기준을 갖지 않도록 단일 소스화
- 우선은 `SHealth.cpp` 익명 namespace 안의 작은 `struct AgeClassPolicy`로 시작
- 이 단계에서는 새 파일 분리 금지, 변경 범위 축소

검증 포인트:
- `20`, `29`, `30`, `70`, `79`, `80` 경계 테스트
- 전체 통계가 `19`, `80`도 포함하는 기존 동작 유지

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

커밋 예시:
- `refactor: centralize age class policy`

### 3. 체중/키 평균 보정 중복 제거
대상:
- `calculateAverageWeights()`
- `calculateAverageHeights()`
- `imputeMissingWeights()`
- `imputeMissingHeights()`

체크리스트:
- 공통 평균 계산 함수 도입:
  - 값 getter 람다를 받는 `calculateAveragesByAgeClass(...)`
- 공통 결측 보정 함수 도입:
  - getter/setter 또는 멤버 포인터 기반
- `weight == 0.0`, `height == 0.0` 정책은 명시적 이름으로 분리
- 외부 동작은 그대로 유지

추천 방향:
- C++17 람다 기반 private helper로 시작
- 템플릿을 쓰더라도 `SHealth.cpp` 내부에 제한
- `std::function`은 불필요하면 피하고 generic lambda/템플릿 선호

검증 포인트:
- weight zero 평균 보정
- height zero 평균 보정
- 같은 연령대 평균이 없을 때 BMI 통계 제외
- 음수 weight/height row skip

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

커밋 예시:
- `refactor: share missing metric imputation logic`

### 4. 비율 집계 중복 제거
대상:
- `calculateRatioSet(int ageClass)`
- `calculateOverallRatioSet()`

체크리스트:
- 공통 `countByCategory(predicate)` 또는 `calculateRatioSetFor(predicate)` 도입
- 연령대별 통계는 predicate로 `toAgeClass(record.age) == ageClass`
- 전체 통계는 predicate로 유효 BMI만 확인
- `counts -> RatioSet` 변환을 별도 함수로 추출
- `record.bmi <= 0.0` 같은 유효 BMI 판단을 `hasValidBmi(record)`로 명명

검증 포인트:
- 연령대별 분모와 전체 분모가 다르게 유지되는지 확인
- invalid type은 계속 `0.0`
- 전체 ratio reset 테스트 유지

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

커밋 예시:
- `refactor: share BMI ratio aggregation`

### 5. CSV 파싱 책임을 내부 헬퍼로 먼저 분리
대상:
- `split()`
- `loadRecords()`
- `parseRecord()`
- CSV delimiter/column index constants

체크리스트:
- 첫 커밋에서는 새 파일보다 `SHealth.cpp` 내부 `CsvHealthRecordParser`/`CsvSchema`로 분리
- `SHealth` 메소드는 `reader.read(filename)` 수준으로 단순화
- `catch (...)`는 가능하면 `std::invalid_argument`, `std::out_of_range`로 좁힘
- row skip 정책은 유지

검증 포인트:
- 컬럼 부족, 빈 숫자, 비숫자, 음수 값 스킵
- ID 보존
- 파일 없음 시 state reset 후 `0` 반환

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

커밋 예시:
- `refactor: isolate CSV record parsing`

### 6. 정책/서비스를 파일 단위로 분리
대상 후보:
- `AgeClassPolicy`
- `BmiTypeMapping`
- `MissingValueImputer`
- `BmiStatisticsService`
- 필요 시 `HealthRecord` 모델

체크리스트:
- 앞 단계에서 안정화된 내부 helper만 새 `.h/.cpp`로 이동
- `CMakeLists.txt`에 새 `.cpp` 추가
- `SHealth`는 facade/application service처럼 흐름만 조율
- public API는 유지
- 전략 패턴은 과도하게 늘리지 말고 변경 가능성이 높은 정책부터 분리:
  - 이미 분리된 `BmiClassifier` 유지
  - `AgeClassPolicy`, `MissingValueImputer`, `BmiStatisticsService`는 우선 값 타입/서비스 객체로 시작
  - 런타임 교체가 필요해질 때만 interface 전략화

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

커밋 예시:
- `refactor: move BMI statistics into service`
- `refactor: move missing value imputation into policy`

### 7. `SHealthBMI.cpp` 출력 반복 제거
대상:
- 20~70대 반복 `printf`
- 100~400 type 반복

체크리스트:
- library의 age class/category mapping과 동일한 테이블을 사용
- `std::cout` + range-based for로 C++17 스타일 정리
- golden master 출력 문자열은 반드시 동일하게 유지하거나, 의도적으로 변경한다면 approved file 갱신을 별도 커밋으로 분리

검증:
```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

커밋 예시:
- `refactor: simplify BMI CLI output loops`

## 진행 원칙

각 커밋은 “한 종류의 리팩토링만” 포함하는 것이 좋습니다. 테스트가 Red가 되면 다음 단계로 가지 않고 해당 커밋 범위 안에서 수정합니다.
