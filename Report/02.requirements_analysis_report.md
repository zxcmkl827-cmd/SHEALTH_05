# SHealth BMI 요구사항 분석 보고서

## 1. 작성 목적

본 문서는 `README.md`에 정의된 SHealth BMI 프로젝트 요구사항을 C++17 구현 및 Google Test 검증 관점에서 재정리한 보고서입니다.

분석 대상은 `SHealth` 클래스의 파일 입력, CSV 파싱, BMI 계산, 누락 체중 보정, 연령대별 BMI 비율 조회 로직입니다. QA 관점에서 정상 동작뿐 아니라 예외 입력, 경계값, 현재 구현의 목적 적합성을 함께 검토합니다.

## 2. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트 | SHealth BMI |
| 언어 표준 | C++17 |
| 빌드 시스템 | CMake |
| 테스트 프레임워크 | Google Test |
| 입력 파일 | `shealth.dat` |
| 주요 데이터 | ID, 나이, 몸무게(kg), 키(cm) |
| 주요 기능 | 누락 체중 보정, BMI 계산, 연령대별 BMI 범주 비율 계산 |

## 3. 파일 입출력 조건

| 항목 | 요구사항/기대 동작 | 현재 C++ 구현 관점 | QA 확인 포인트 |
|---|---|---|---|
| 입력 파일 경로 | `calculateBmi(filename)` 호출자가 CSV 데이터 파일 경로를 전달한다. 기본 실행 파일은 `shealth.dat`를 사용한다. | `SHealthBMI.cpp`에서 상대 경로 `"shealth.dat"`를 전달한다. 실행 위치가 프로젝트 루트가 아니면 파일을 찾지 못할 수 있다. | 실행 디렉터리별 상대 경로 동작 확인이 필요하다. |
| 파일 존재 여부 | 파일이 없으면 계산을 수행하지 않고 실패를 명확히 처리해야 한다. | `std::ifstream file(filename)` 후 `is_open()` 실패 시 `std::cerr` 출력 후 `0`을 반환한다. | 반환값 `0`과 오류 로그를 검증한다. 기존 ratio 값 초기화 여부도 확인한다. |
| 파일 open 여부 | 파일 open 실패, 권한 없음, 디렉터리 경로 입력 등을 처리해야 한다. | open 실패는 모두 동일하게 `0`을 반환한다. 실패 사유는 구분하지 않는다. | 존재하지 않는 파일, 읽기 권한 없는 파일, 디렉터리 경로 입력 테스트가 필요하다. |
| 빈 파일 | 헤더도 없는 파일은 데이터 0건으로 처리되어야 한다. | 첫 줄을 헤더로 읽은 뒤 데이터 루프를 수행하지 않는다. 이후 연령대별 비율 계산에서 `sum == 0`으로 나눗셈이 발생할 수 있다. | 빈 파일 또는 헤더만 있는 파일에서 NaN/Inf 발생 여부를 검증한다. |
| 최대 데이터 건수 | 입력 건수 제한이 있으면 명확히 정의하고 초과 시 안전하게 처리해야 한다. | `ages`, `weights`, `heights`, `bmis`가 고정 배열 10,000개이다. 초과 입력에 대한 검사 없이 `count++` 한다. | 10,000건 초과 입력 시 메모리 오염 위험이 있다. |
| 파일 close | 정상 처리 후 파일 리소스는 해제되어야 한다. | `file.close()`를 명시 호출한다. RAII만으로도 가능하지만 현재 동작상 문제는 작다. | 예외 발생 시 close까지 도달하지 않을 수 있으나 `ifstream` 소멸자가 정리한다. |

## 4. 파일 Read Format

| 항목 | 요구사항/기대 동작 | 현재 C++ 구현 관점 | QA 확인 포인트 |
|---|---|---|---|
| 파일 형식 | CSV 텍스트 파일이며 1행은 헤더, 2행부터 데이터이다. | 첫 줄을 무조건 헤더로 읽고 검증하지 않는다. | 헤더 누락, 헤더명 변경, 헤더 없이 데이터가 시작되는 경우를 구분해야 한다. |
| 컬럼 순서 | `id,age,weight,height` 순서이다. | `tokens[1]`, `tokens[2]`, `tokens[3]`만 사용하며 ID는 저장/검증하지 않는다. | ID 누락/중복/비숫자 여부는 현재 계산에 영향이 없지만 데이터 품질 요구사항으로 정의가 필요하다. |
| split 기준 | 쉼표(`,`) 기준으로 필드를 분리한다. | `split(line, ',')`가 `std::getline` 기반으로 단순 분리한다. 따옴표, 이스케이프, 쉼표 포함 값은 지원하지 않는다. | CSV quoting 미지원 범위를 명시하거나 표준 CSV 파서를 고려해야 한다. |
| 공백 처리 | 숫자 주변 공백은 허용 여부를 정의해야 한다. | `stoi`, `stod`는 선행 공백은 처리 가능하지만 필드 전체 검증은 하지 않는다. | `" 20"`, `"20 "`, `"20abc"` 같은 입력의 허용 여부를 정해야 한다. |
| 누락 컬럼 | 컬럼 수가 4개 미만이면 해당 라인을 거부하거나 오류 처리해야 한다. | `tokens[1..3]`에 직접 접근한다. 컬럼 부족 시 범위 밖 접근으로 비정상 동작 가능성이 있다. | `93705,66,79.5`, `93705,66,,158.3` 등을 검증한다. |
| 추가 컬럼 | 4개 초과 컬럼 허용 여부를 정의해야 한다. | 5번째 이후 컬럼은 무시한다. | 추가 컬럼이 있어도 처리할지, 오류로 볼지 결정해야 한다. |
| 빈 줄 | 빈 줄은 무시하거나 오류로 처리해야 한다. | 빈 줄은 `tokens.empty()`가 되어 전체 read 루프를 `break`한다. 이후 유효한 라인이 있어도 읽지 않는다. | 중간 빈 줄이 전체 데이터 절단을 일으키는지 확인한다. |
| 체중 누락값 | README 기준 체중 `0`은 누락값이며 같은 나이대 평균 체중으로 보정한다. | 20대부터 70대까지 10년 단위 그룹에서 `weight == 0.0`만 평균으로 대체한다. | 0이 아닌 음수, 빈 값, 비숫자 값은 누락으로 처리되지 않는다. |
| 키 누락값 | README에는 키 누락 보정이 초기 요구사항이 아니며 Activities에는 개선 항목으로 언급된다. | `height == 0` 보정 로직이 없다. BMI 계산 시 0으로 나눗셈이 발생한다. | 키 0은 현재 결함/개선 대상이다. |

## 5. 예외 및 경계값 조건

| 조건 | 기대 처리 | 현재 구현 리스크 | 우선순위 |
|---|---|---|---|
| 나이 음수 | 입력 오류로 거부하거나 통계 대상에서 제외해야 한다. | `count`에는 포함되지만 20~79세 통계에는 제외된다. 명시적 오류가 없다. | 중 |
| 나이 0~19 | 통계 대상 제외 또는 별도 연령대 정책이 필요하다. | BMI는 계산하지만 비율에는 반영하지 않는다. | 중 |
| 나이 20, 29, 30, 79 | 10년 단위 그룹 경계값을 정확히 포함해야 한다. | `[20,30)`, `[30,40)`, ..., `[70,80)`로 처리된다. | 높음 |
| 나이 80 이상 | 포함 여부를 명확히 정의해야 한다. | 통계에는 제외된다. | 중 |
| 체중 0 | 같은 나이대의 0이 아닌 체중 평균으로 보정한다. | 해당 연령대의 모든 체중이 0이면 `sum / ageCount`에서 0으로 나눗셈이 발생한다. | 높음 |
| 체중 음수 | 입력 오류로 처리해야 한다. | 음수 BMI가 계산되어 저체중으로 분류될 수 있다. | 높음 |
| 체중 비숫자/빈 값 | 라인 오류로 처리하거나 파일 오류로 실패해야 한다. | `std::stod` 또는 `std::stoi`에서 예외가 발생할 수 있다. 현재 catch 없음. | 높음 |
| 키 0 | 입력 오류 또는 평균 키 보정 정책이 필요하다. | BMI 계산 시 0으로 나눗셈이 발생한다. | 높음 |
| 키 음수 | 입력 오류로 처리해야 한다. | 제곱 계산 때문에 양수 키와 동일한 BMI가 계산될 수 있다. | 높음 |
| 키 비숫자/빈 값 | 라인 오류로 처리하거나 파일 오류로 실패해야 한다. | `std::stod` 예외가 발생할 수 있다. 현재 catch 없음. | 높음 |
| BMI 18.5 | README 기준 저체중이다. | `bmi <= 18.5`로 저체중 처리한다. | 높음 |
| BMI 23.0 | README 기준 과체중이다. | `bmi >= 23 && bmi < 25`로 과체중 처리한다. | 높음 |
| BMI 25.0 | README 기준 비만이다. | `bmi > 25`만 비만으로 처리하므로 정확히 25.0은 어느 범주에도 포함되지 않는다. | 높음 |
| 연령대 데이터 0건 | 해당 연령대 비율은 0 또는 미계산 상태로 정의되어야 한다. | `sum == 0`이면 비율 계산에서 0으로 나눗셈이 발생한다. | 높음 |
| 파일 재계산 호출 | 같은 객체에서 재호출해도 이전 결과가 남지 않아야 한다. | `count`만 초기화하고 ratio 필드는 파일 open 실패 시 이전 값이 남을 수 있다. | 중 |

## 6. Google Test 기준 테스트 시나리오

| 번호 | 테스트 시나리오 | 검증 대상 |
|---:|---|---|
| TC-001 | 정상 CSV 파일을 읽으면 `calculateBmi`가 데이터 건수를 반환한다. | 파일 read, count 반환 |
| TC-002 | 존재하지 않는 파일 경로를 전달하면 `0`을 반환한다. | 파일 open 실패 처리 |
| TC-003 | 헤더만 있는 파일을 입력하면 비율 계산이 안전하게 처리된다. | 데이터 0건, 0 나눗셈 방지 |
| TC-004 | 중간에 빈 줄이 있는 파일에서 이후 데이터 처리 정책을 검증한다. | 빈 줄 처리 |
| TC-005 | 컬럼이 4개 미만인 라인을 입력하면 안전하게 실패하거나 스킵한다. | 누락 컬럼 처리 |
| TC-006 | 비숫자 나이/체중/키가 포함된 라인을 입력하면 예외 정책대로 처리된다. | `stoi`, `stod` 예외 처리 |
| TC-007 | 체중이 0인 사용자는 같은 나이대 평균 체중으로 보정된다. | 누락 체중 보정 |
| TC-008 | 같은 나이대의 모든 체중이 0일 때 안전하게 처리된다. | 평균 계산 분모 0 |
| TC-009 | 키가 0인 데이터는 안전하게 오류 처리되거나 보정된다. | BMI 계산 분모 0 |
| TC-010 | 음수 나이 입력 시 정책대로 제외 또는 오류 처리된다. | 나이 유효성 |
| TC-011 | 음수 체중 입력 시 정책대로 오류 처리된다. | 체중 유효성 |
| TC-012 | 음수 키 입력 시 정책대로 오류 처리된다. | 키 유효성 |
| TC-013 | 나이 20, 29, 30, 39, 70, 79의 연령대 분류가 정확하다. | 연령대 경계값 |
| TC-014 | 나이 19와 80 입력 시 통계 제외 또는 별도 정책이 적용된다. | 지원 연령 범위 |
| TC-015 | BMI 18.5는 저체중으로 분류된다. | BMI 하한 경계 |
| TC-016 | BMI가 18.5 초과 23 미만이면 정상체중으로 분류된다. | 정상 범위 |
| TC-017 | BMI 23.0은 과체중으로 분류된다. | 과체중 경계 |
| TC-018 | BMI 25.0은 비만으로 분류된다. | 비만 경계 |
| TC-019 | 특정 연령대에 데이터가 없으면 `getBmiRatio`가 안전한 값을 반환한다. | 비율 분모 0 |
| TC-020 | 알 수 없는 `ageClass` 또는 `type`은 `0.0`을 반환한다. | 조회 API 기본값 |
| TC-021 | 같은 `SHealth` 객체로 정상 파일 계산 후 실패 파일을 계산해도 이전 ratio가 노출되지 않는다. | 객체 상태 초기화 |
| TC-022 | 10,000건 초과 입력에 대해 안전하게 제한 또는 오류 처리된다. | 고정 배열 한계 |

## 7. 구현된 메소드 목적 적합성

| 메소드 | 목적 | 현재 적합성 | 개선 필요 사항 |
|---|---|---|---|
| `calculateBmi(const std::string& filename)` | 파일을 읽고, 누락 체중을 보정하고, BMI를 계산하고, 연령대별 BMI 범주 비율을 산출한다. | 부분 적합. 정상 데이터와 20~79세 범위에서는 핵심 흐름을 수행한다. | 파일 파싱 검증, 예외 처리, 분모 0 방지, BMI 25.0 분류 수정, 상태 초기화, 고정 배열 제거가 필요하다. |
| `getBmiRatio(int ageClass, int type)` | 특정 연령대와 BMI 범주 코드에 해당하는 비율을 반환한다. | 부분 적합. 20~70 연령대와 100/200/300/400 타입 코드를 조회할 수 있다. | 매직 넘버를 enum/class로 대체하고, 미지원 입력 정책을 명확히 해야 한다. |
| `split(const std::string& line, char delimiter)` | CSV 라인을 구분자로 분리한다. | 제한적 적합. 단순 쉼표 구분 데이터에는 동작한다. | trailing empty field, quoting, 컬럼 수 검증, 공백/빈 값 정책을 보완해야 한다. |

## 8. 종합 의견

1. README의 핵심 요구사항인 체중 0 보정, BMI 계산, 연령대별 BMI 범주 비율 산출은 현재 구현에 포함되어 있다.
2. 정상 데이터 기준의 기본 흐름은 동작 가능하지만, 입력 검증과 예외 처리 수준은 QA 기준에서 부족하다.
3. 가장 우선적으로 수정해야 할 결함은 `BMI 25.0` 미분류, `sum == 0` 및 `ageCount == 0` 나눗셈, 컬럼 누락/비숫자 입력 예외, `height == 0`, 10,000건 초과 배열 접근이다.
4. 현재 Google Test는 실패 테스트 1개만 존재하므로, 본 보고서의 TC-001~TC-022를 기준으로 정상/예외/경계값 테스트를 먼저 구축하는 것이 필요하다.
