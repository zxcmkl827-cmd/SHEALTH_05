# SHealth Modern C++ Refactoring Report

## 1. 작성 목적

본 문서는 `src/main/cpp/SHealth.h`, `src/main/cpp/SHealth.cpp`를 중심으로 수행한 Modern C++ 리팩토링 결과를 정리한 보고서입니다.

리팩토링 목표는 기존 BMI 계산 로직의 조건 분기와 중복을 줄이고, 고정 배열과 매직 넘버 중심의 구현을 C++17 스타일의 타입, 컨테이너, 정책 객체 기반 구조로 개선하는 것입니다. 추가로 BMI 타입 판단 로직을 인터페이스와 다형성 기반 전략으로 분리하여 향후 BMI 기준 변경에 대응할 수 있도록 했습니다.

## 2. 리팩토링 전 주요 문제

### 2.1 `SHealth::calculateBmi`의 과도한 책임

기존 `calculateBmi`는 하나의 메소드에서 다음 책임을 모두 처리했습니다.

- 파일 열기
- CSV 라인 분리
- 문자열을 숫자로 변환
- 배열에 데이터 저장
- 누락 체중 보정
- BMI 계산
- 나이대별 BMI 범주 비율 계산
- 결과 멤버 변수 저장

이 구조는 SRP를 위반하고, 파일 포맷 변경이나 BMI 기준 변경이 모두 같은 메소드 수정을 유발했습니다.

### 2.2 고정 배열과 분산된 사용자 데이터

기존 구현은 `ages[10000]`, `weights[10000]`, `heights[10000]`, `bmis[10000]`처럼 병렬 배열을 사용했습니다.

이 방식은 한 사람의 건강 데이터가 여러 배열에 흩어져 데이터 응집도가 낮고, 10,000건 초과 입력에 대한 범위 검사가 없어 메모리 접근 위험이 있었습니다.

### 2.3 조건문 체인과 중복된 비율 멤버

기존 `getBmiRatio`는 `ageClass`와 `type` 조합을 24개의 `if/else if`로 처리했습니다. 또한 `underweight20`, `normalweight20`, `overweight20`, `obesity20`처럼 나이대와 BMI 범주 조합을 멤버 변수명으로 표현했습니다.

이 구조는 새 나이대나 새 BMI 타입이 추가될 때 조건문과 멤버 변수가 함께 증가하는 구조였습니다.

### 2.4 BMI 기준의 하드코딩

BMI 기준값 `18.5`, `23`, `25`와 타입 코드 `100`, `200`, `300`, `400`이 계산 로직 안에 직접 사용되었습니다.

특히 기존 코드는 `bmi > 25` 조건으로 비만을 판단해 README 요구사항상 비만이어야 하는 `BMI == 25.0`이 어떤 범주에도 포함되지 않는 문제가 있었습니다.

## 3. 1차 리팩토링 결과

### 3.1 데이터 구조 개선

사용자 건강 데이터를 `HealthRecord` 구조체로 묶고, `std::vector<HealthRecord>`로 관리하도록 변경했습니다.

변경 후 한 사용자의 `age`, `weight`, `height`, `bmi`가 하나의 값 타입으로 묶이므로 데이터 응집도가 높아졌고, 고정 배열 크기 제한도 제거되었습니다.

### 3.2 비율 저장 구조 개선

나이대별 BMI 비율은 `std::map<int, RatioSet>`으로 저장하도록 변경했습니다.

`RatioSet`은 BMI 범주별 비율을 `std::array<double, 4>`로 보관합니다. 이로써 나이대별 멤버 변수 조합과 `if/else if` 저장 로직을 제거했습니다.

### 3.3 `calculateBmi` 오케스트레이터화

현재 `calculateBmi`는 전체 흐름만 조율합니다.

```cpp
int SHealth::calculateBmi(const std::string& filename) {
    resetState();

    auto loadedRecords = loadRecords(filename);
    if (!loadedRecords) {
        return 0;
    }

    records = std::move(*loadedRecords);
    imputeMissingWeights();
    calculateBmiValues();
    ratiosByAgeClass = aggregateRatios();

    return static_cast<int>(records.size());
}
```

세부 책임은 다음 메소드로 분리했습니다.

- `loadRecords`: 파일 열기 및 레코드 로딩
- `parseRecord`: CSV 라인 파싱 및 숫자 변환
- `calculateAverageWeights`: 나이대별 유효 체중 평균 계산
- `imputeMissingWeights`: 체중 `0` 결측값 보정
- `calculateBmiValues`: BMI 계산
- `aggregateRatios`: 나이대별 BMI 비율 집계
- `calculateRatioSet`: 특정 나이대의 BMI 범주별 비율 계산

### 3.4 파일 입력 조건 강화

파일 입력과 파싱에서 다음 조건을 보강했습니다.

- 파일 open 실패 시 `std::nullopt` 반환
- 빈 줄은 전체 처리를 중단하지 않고 무시
- CSV 컬럼이 부족한 라인은 무시
- `stoi`, `stod` 변환 예외는 `parseRecord`에서 처리
- 음수 체중은 무효 레코드로 처리
- 키 `0` 이하 값은 BMI 계산에서 제외되도록 무효 레코드로 처리
- 비율 계산 시 분모가 `0`이면 `0.0` 반환

## 4. BMI 타입 추상화 및 다형성 적용

### 4.1 분리한 타입과 클래스

BMI 타입 판단 책임을 `SHealth`에서 분리하기 위해 다음 파일을 추가했습니다.

- `src/main/cpp/BmiCategory.h`
- `src/main/cpp/BmiClassifier.h`
- `src/main/cpp/StandardBmiClassifier.h`
- `src/main/cpp/StandardBmiClassifier.cpp`

`BmiCategory`는 BMI 범주를 표현하는 enum입니다.

```cpp
enum class BmiCategory {
    Underweight = 0,
    Normal = 1,
    Overweight = 2,
    Obesity = 3,
};
```

`BmiClassifier`는 BMI 수치를 BMI 범주로 분류하는 인터페이스입니다.

```cpp
class BmiClassifier {
public:
    virtual ~BmiClassifier() = default;
    virtual BmiCategory classify(double bmi) const = 0;
};
```

`StandardBmiClassifier`는 README 기준을 구현하는 기본 정책 클래스입니다.

```cpp
BmiCategory StandardBmiClassifier::classify(double bmi) const {
    if (bmi <= UnderweightMax) {
        return BmiCategory::Underweight;
    }
    if (bmi < NormalMax) {
        return BmiCategory::Normal;
    }
    return bmi < OverweightMax ? BmiCategory::Overweight : BmiCategory::Obesity;
}
```

### 4.2 `SHealth`의 전략 주입 구조

`SHealth`는 이제 `std::unique_ptr<BmiClassifier>`를 멤버로 보관합니다.

기본 생성자는 `StandardBmiClassifier`를 사용하고, 필요하면 외부에서 다른 BMI 분류 정책을 주입할 수 있습니다.

```cpp
SHealth::SHealth() : SHealth(std::make_unique<StandardBmiClassifier>()) {
}

SHealth::SHealth(std::unique_ptr<BmiClassifier> classifier)
    : bmiClassifier(std::move(classifier)) {
    if (!bmiClassifier) {
        bmiClassifier = std::make_unique<StandardBmiClassifier>();
    }
}
```

나이대별 비율 집계에서는 더 이상 `SHealth` 내부 분류 함수가 아니라 주입된 정책 객체를 사용합니다.

```cpp
counts[static_cast<std::size_t>(bmiClassifier->classify(record.bmi))]++;
```

이 변경으로 `SHealth`는 BMI 분류 기준을 직접 알 필요가 없어졌습니다. 향후 기관별 기준, 국가별 기준, 테스트용 기준을 추가할 때 `BmiClassifier` 구현체만 추가하면 됩니다.

## 5. 현재 파일별 변경 요약

### 5.1 `src/main/cpp/SHealth.h`

- 파싱, 보정, 계산, 집계 헬퍼 메소드 선언 추가

### 5.2 `src/main/cpp/SHealth.cpp`

- `calculateBmi`를 전체 처리 흐름만 담당하는 메소드로 축소
- 파일 로딩과 CSV 파싱을 `loadRecords`, `parseRecord`로 분리
- 체중 보정을 `calculateAverageWeights`, `imputeMissingWeights`로 분리
- BMI 계산을 `calculateBmiValue`, `calculateBmiValues`로 분리

- BMI 분류는 `bmiClassifier->classify()` 호출로 변경

### 5.4 `src/main/cpp/BmiClassifier.h`

### 5.5 `src/main/cpp/StandardBmiClassifier.h/.cpp`

- README의 기본 BMI 기준 구현

### 5.6 `CMakeLists.txt`

## 6. 설계 관점 평가

### 6.1 SRP 개선

기존에는 `SHealth::calculateBmi` 하나가 파일 처리, 데이터 정제, BMI 계산, 통계 집계, 분류 기준까지 모두 담당했습니다.

현재는 각 책임이 작은 private 메소드와 BMI 정책 클래스로 나뉘었습니다. 특히 BMI 분류 기준은 `StandardBmiClassifier`로 이동해 `SHealth`의 변경 이유를 줄였습니다.

### 6.2 OCP 개선

BMI 기준 변경이 필요한 경우 기존에는 `SHealth.cpp`의 조건문을 직접 수정해야 했습니다.

현재는 `BmiClassifier`를 상속한 새 구현체를 추가하고 `SHealth`에 주입하면 됩니다. 따라서 기존 `SHealth` 로직을 수정하지 않고도 새로운 BMI 정책을 확장할 수 있습니다.

### 6.3 다형성 적용 효과

다형성 적용 전에는 BMI 범주 판단이 `SHealth::classifyBmi`에 고정되어 있었습니다.

다형성 적용 후에는 런타임에 주입된 `BmiClassifier` 구현체가 분류 기준을 결정합니다. 테스트에서는 fake classifier를 만들어 모든 BMI를 특정 범주로 분류하게 하거나, 다른 경계값을 가진 classifier를 주입할 수 있습니다.

### 6.4 C++17 스타일 개선

이번 리팩토링에는 다음 C++17 스타일 요소가 반영되었습니다.

- `std::vector` 기반 동적 레코드 관리
- `std::map` 기반 나이대별 통계 저장
- `std::array` 기반 고정 범주 비율 저장
- `std::optional` 기반 파일 로딩 및 파싱 실패 표현
- `std::unique_ptr` 기반 전략 객체 소유권 관리
- `constexpr` 기반 매직 넘버 상수화
- 구조적 바인딩을 사용한 평균 계산

## 8. 남은 리스크 및 후속 작업

### 8.1 테스트 교체 필요

현재 `SHealthBMITest.cpp`에는 `FAIL()`만 존재합니다. 리팩토링 안정성을 보장하려면 다음 테스트로 교체해야 합니다.

- 정상 CSV 입력 시 `calculateBmi`가 레코드 수를 반환하는지 검증
- BMI `18.5`, `23.0`, `25.0` 경계값 검증
- 체중 `0` 보정 검증
- 빈 파일 또는 헤더만 있는 파일에서 비율이 `0.0`인지 검증
- 잘못된 파일 경로에서 `0` 반환 및 상태 초기화 검증
- fake `BmiClassifier` 주입 시 집계 결과가 주입 정책을 따르는지 검증

### 8.2 CSV 파싱 정책 명확화

현재 CSV 파싱은 단순 쉼표 분리 방식입니다. 따옴표, 이스케이프, 쉼표 포함 필드는 지원하지 않습니다.

입력 데이터가 단순 CSV라는 요구사항이면 현재 방식으로 충분하지만, 표준 CSV 호환이 필요하면 CSV 파서를 별도 클래스로 분리하거나 검증된 파서를 도입하는 것이 좋습니다.

### 8.3 오류 처리 정책 고도화

현재 `calculateBmi`는 파일 open 실패와 유효 데이터 0건을 모두 `0` 반환으로 표현합니다.

실패 원인을 호출자가 구분해야 한다면 `std::optional<int>`, 오류 코드 enum, 또는 결과 객체를 도입하는 후속 개선이 필요합니다.

### 8.4 빌드 산출물 관리

검증 과정에서 `build/` 디렉터리가 생성되었고 현재 Git 기준 untracked로 표시됩니다.

프로젝트 정책에 따라 `build/`는 `.gitignore`에 추가하거나, 커밋 대상에서 제외해야 합니다.

## 9. 결론

이번 리팩토링을 통해 `SHealth`는 파일 로딩, 레코드 파싱, 결측값 보정, BMI 계산, 통계 집계의 흐름을 작은 메소드로 나누게 되었습니다.

또한 BMI 타입 판단은 `BmiClassifier` 인터페이스와 `StandardBmiClassifier` 구현체로 분리되어, BMI 기준 변경이 `SHealth`의 핵심 집계 로직에 영향을 주지 않도록 개선되었습니다.

현재 코드는 `cmake --build build` 기준으로 빌드 가능하며, `ctest`는 기존 의도적 실패 테스트를 실제 검증 테스트로 교체한 뒤 통과 여부를 다시 확인해야 합니다.
