# SHEALTH BMI 테스트 계획 작성 보고서

## 1. 작성 목적

본 문서는 SHEALTH BMI 모듈의 요구사항 분석 결과와 현재 C++17 구현 상태를 바탕으로 테스트 계획서를 작성한 내용을 정리한 보고서입니다.

시니어 QA 리드 관점에서 `TEST_F` 기반 Google Test 단위 테스트 범위, 경계값 테스트, 예외/특이 케이스, gcov/lcov 커버리지 목표와 측정 전략을 문서화하는 것을 목표로 했습니다.

## 2. 작업 대상

- 요구사항 분석 문서: `docs/requirements_analysis.md`
- 테스트 계획서: `docs/test_plan.md`
- 주요 구현 파일:
  - `src/main/cpp/SHealth.cpp`
  - `src/main/cpp/SHealthBMI.cpp`

- 테스트 파일: `src/test/cpp/SHealthBMITest.cpp`
- 빌드 설정: `CMakeLists.txt`

## 3. 확인한 현재 상태

현재 프로젝트는 CMake 기반으로 `shealth_lib`, `SHealthBMI`, `SHealthBMITest` 타깃이 분리되어 있습니다. Google Test `v1.14.0`은 `FetchContent`로 자동 다운로드되며, `gtest_discover_tests(SHealthBMITest)`를 통해 CTest에 연결됩니다.

`SHealth.cpp`는 다음 흐름을 중심으로 구성되어 있습니다.

- CSV 파일 로드
- 레코드 파싱
- 체중 `0` 누락값의 같은 연령대 평균 보정
- BMI 계산
- 20대부터 70대까지 10년 단위 BMI 범주 비율 산출
- 알 수 없는 연령대 또는 BMI 타입 조회 시 `0.0` 반환

`StandardBmiClassifier.cpp`는 BMI 기준을 독립 분류기로 분리하고 있으며, `18.5`, `23.0`, `25.0` 경계값 검증이 중요한 테스트 포인트입니다.

현재 `src/test/cpp/SHealthBMITest.cpp`에는 placeholder 실패 테스트만 존재하므로, 실제 테스트 자동화는 아직 시작 전 상태입니다.

## 4. 작성한 테스트 계획서 주요 내용

`docs/test_plan.md`에 다음 내용을 정리했습니다.

- `SHealthTest`와 `StandardBmiClassifierTest`로 fixture를 분리하는 `TEST_F` 기반 테스트 구조
- P0, P1, P2 우선순위별 단위 테스트 범위
- 나이, 체중, 키, BMI 분류 기준의 경계값 케이스
- 입력 파일 없음, 빈 파일, 헤더만 있는 파일, 컬럼 부족, 비숫자 입력, 중간 빈 줄 등 예외/특이 케이스
- `SHealth.cpp` 95% 이상, 전체 라인 커버리지 90% 이상, 분기 커버리지 85% 이상 목표
- GCC/Clang 기반 gcov/lcov 측정 명령과 커버리지 개선 루프
- Windows/MSVC 환경에서는 별도 GCC/Clang 커버리지 빌드를 권장하는 운영 전략

## 5. 테스트 우선순위 요약

### P0

- 정상 CSV 입력 시 유효 레코드 수 반환
- 연령대별 저체중/정상/과체중/비만 비율 산출
- 체중 `0` 누락값의 같은 연령대 평균 보정
- 알 수 없는 연령대 또는 BMI 타입 조회 시 `0.0` 반환
- 같은 `SHealth` 객체 반복 호출 시 이전 상태 제거

### P1

- BMI `18.5`, `23.0`, `25.0` 경계값 분류
- 나이 `20`, `29`, `30`, `39`, `70`, `79` 연령대 경계값
- 입력 파일 없음
- 컬럼 부족, 비숫자 입력, 음수 체중, 키 `0` 또는 음수
- 중간 빈 줄 처리

### P2

- 빈 파일과 헤더만 있는 파일
- 같은 연령대 체중이 모두 `0`인 경우
- 추가 컬럼 무시 정책
- 숫자 주변 공백 처리
- trailing empty field 처리

## 6. 주요 QA 리스크

- 음수 나이는 현재 파싱 단계에서 거부되지 않고 비율 집계에서만 제외됩니다. 입력 오류로 볼지, 통계 제외로 볼지 정책 확정이 필요합니다.
- 헤더가 없는 CSV 파일은 첫 번째 데이터 행이 헤더로 소비될 수 있습니다. 헤더 검증 정책이 필요합니다.
- 단순 쉼표 split 기반이므로 표준 CSV quoting, escape, 쉼표 포함 값은 지원하지 않습니다.
- 체중 `0` 보정은 같은 연령대에 유효 체중이 있을 때만 가능합니다.
- 키 누락값 보정은 현재 구현 범위 밖이며, README의 개선 항목으로 남아 있습니다.
- 실제 Google Test 자동화는 아직 placeholder 실패 테스트를 대체해야 하는 상태입니다.

## 7. 커버리지 전략

커버리지 목표는 전체 라인 90% 이상, `SHealth.cpp` 95% 이상, `StandardBmiClassifier.cpp` 100%, 분기 커버리지 85% 이상으로 설정했습니다.

gcov/lcov 측정은 MSVC 빌드와 직접 호환되지 않으므로 GCC 또는 Clang 기반 별도 커버리지 빌드를 권장합니다. 일반 Debug/Release 빌드와 분리하기 위해 `ENABLE_COVERAGE` 같은 CMake 옵션을 추가하는 방안을 테스트 계획서에 포함했습니다.

개선 순서는 다음과 같습니다.

1. P0 테스트로 정상 흐름과 상태 초기화를 먼저 고정합니다.
2. `StandardBmiClassifier` 경계값 테스트로 BMI 분류 분기를 100% 커버합니다.
3. 파일 open 실패, 잘못된 행, 빈 파일, 헤더만 있는 파일로 파싱 방어 로직을 검증합니다.
4. lcov 리포트에서 남은 미커버 라인이 요구사항상 의미 있는 동작인지 판단합니다.
5. 의미 있는 미커버 라인은 테스트를 추가하고, 의미 없는 방어 코드는 구조 개선 후보로 분류합니다.
