# SHEALTH BMI Google Test 구현 보고서

## 1. 작성 목적

본 문서는 SHEALTH BMI 모듈에 대해 C++17, Google Test 기반 테스트 코드를 구현한 내용을 정리한 보고서입니다.

요구사항은 BMI 타입별 최소 5개 `TEST_F`를 작성하고, 저체중/정상체중/과체중/비만 각각에 대해 경계값과 예외 입력을 포함하며, `EXPECT_EQ` 또는 `ASSERT_EQ`로 BMI 결과를 검증하는 것이었습니다.

## 2. 작업 대상

- 테스트 파일: `src/test/cpp/SHealthBMITest.cpp`
- 주요 구현 파일:
  - `src/main/cpp/SHealth.h`
  - `src/main/cpp/SHealth.cpp`
  - `src/main/cpp/BmiClassifier.h`
  - `src/main/cpp/BmiCategory.h`
  - `src/main/cpp/StandardBmiClassifier.cpp`
- 요구사항 분석 문서: `docs/requirements_analysis.md`
- 빌드 설정: `CMakeLists.txt`

## 3. 구현 전 상태

기존 `src/test/cpp/SHealthBMITest.cpp`에는 다음과 같은 placeholder 실패 테스트만 존재했습니다.

```cpp
TEST(SHealthBMITest, FailedTest) {
    FAIL();
}
```

따라서 `cmake --build build` 이후 `ctest`를 실행하면 테스트가 실패할 수밖에 없는 상태였습니다.

## 4. 구현한 테스트 구조

`SHealthBMITest` fixture를 추가하고, 테스트마다 임시 CSV 파일을 생성해 `SHealth::calculateBmi()`와 `SHealth::getBmiRatio()`를 검증하도록 구성했습니다.

공통 헬퍼는 다음 역할을 수행합니다.

- `writeCsv()`: 테스트별 임시 CSV 파일 생성
- `expectOnlyRatio()`: 특정 연령대에서 기대 BMI 타입만 `100.0`, 나머지는 `0.0`인지 검증

BMI 타입 코드는 구현의 public 조회 API에 맞춰 다음 상수로 정의했습니다.

- 저체중: `100`
- 정상체중: `200`
- 과체중: `300`
- 비만: `400`

모든 테스트는 `Given-When-Then` 주석 구조를 적용했습니다.

## 5. BMI 타입별 테스트 케이스

### 저체중

저체중은 BMI `18.5` 이하 기준과 연령대 경계, 음수 나이 제외 동작을 포함해 5개 `TEST_F`로 검증했습니다.

- `UnderweightWhenBmiIsBelowLowerBoundary`
- `UnderweightWhenBmiEqualsLowerBoundary`
- `UnderweightKeepsAgeThirtyBoundaryInThirtyClass`
- `UnderweightKeepsAgeSeventyNineInSeventyClass`
- `UnderweightExcludesNegativeAgeFromRatios`

### 정상체중

정상체중은 BMI `18.5` 초과, `23.0` 미만 범위와 체중 `0` 보정, 음수 몸무게 스킵 동작을 포함해 5개 `TEST_F`로 검증했습니다.

- `NormalWhenBmiIsJustAboveUnderweightBoundary`
- `NormalWhenBmiIsInMiddleOfRange`
- `NormalWhenBmiIsJustBelowOverweightBoundary`
- `NormalUsesImputedWeightFromSameAgeClass`
- `NormalSkipsNegativeWeightRecord`

### 과체중

과체중은 BMI `23.0` 이상, `25.0` 미만 범위와 음수 키, 키 `0` 입력 스킵 동작을 포함해 5개 `TEST_F`로 검증했습니다.

- `OverweightWhenBmiEqualsLowerBoundary`
- `OverweightWhenBmiIsInMiddleOfRange`
- `OverweightWhenBmiIsJustBelowObesityBoundary`
- `OverweightSkipsNegativeHeightRecord`
- `OverweightSkipsZeroHeightRecord`

### 비만

비만은 BMI `25.0` 이상 기준과 나이 `70`, 나이 `80` 경계 동작을 포함해 5개 `TEST_F`로 검증했습니다.

- `ObesityWhenBmiEqualsLowerBoundary`
- `ObesityWhenBmiIsAboveLowerBoundary`
- `ObesityWhenHeightIsTwoMetersAndWeightIsOneHundred`
- `ObesityKeepsAgeSeventyBoundaryInSeventyClass`
- `ObesityExcludesAgeEightyFromRatios`

## 6. 주요 검증 포인트

이번 테스트 구현에서 중점적으로 확인한 항목은 다음과 같습니다.

- `calculateBmi()`가 유효하게 처리한 레코드 수를 `ASSERT_EQ`로 검증
- `getBmiRatio(ageClass, type)` 결과를 `EXPECT_EQ`로 검증
- BMI `18.5`, `23.0`, `25.0` 경계값 검증
- 나이 `30`, `70`, `79`, `80`, 음수 나이 경계 검증
- 체중 `0` 누락값의 같은 연령대 평균 보정 검증
- 음수 몸무게, 음수 키, 키 `0` 레코드 스킵 검증
- 각 BMI 타입별 최소 5개 `TEST_F` 충족

## 7. 검증 결과

편집 후 `src/test/cpp/SHealthBMITest.cpp`에 대해 linter 확인을 수행했으며, 오류는 발견되지 않았습니다.

빌드 디렉터리가 없는 상태였으므로 먼저 CMake 구성을 수행했습니다.

```powershell
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
```

검증 결과는 다음과 같습니다.

- 빌드 성공
- CTest 전체 20개 테스트 실행
- 20개 테스트 모두 통과
- 실패 테스트 0개

추가로 `build` 디렉터리에서 `ctest --output-on-failure`를 단독 실행했으며, 동일하게 20개 테스트가 모두 통과했습니다.

## 8. 주의 사항

사용자가 요청한 `cmake --build build && ctest` 형태의 명령은 현재 PowerShell 환경에서 `&&` 토큰을 지원하지 않아 파싱 오류가 발생했습니다.

따라서 같은 의미의 조건부 실행을 PowerShell 문법으로 변환해 검증했습니다.

```powershell
cmake --build build
ctest --test-dir build --output-on-failure
```

최종적으로 빌드와 테스트는 Green 상태입니다.
