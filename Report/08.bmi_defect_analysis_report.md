# SHEALTH BMI 디버깅 및 결함 분석 보고서

## 1. 작성 목적

본 문서는 BMI 테스트 실패 로그 분석 요청에 대해 현재 코드와 테스트 실행 결과를 기준으로 `EXPECT_EQ` 실패 여부, `SHealth::calculateBmi()`, `SHealth::getBmiRatio()` 관련 결함 위치, 결함 심각도, 최소 변경 수정 방안, Green 확인 절차를 정리한 보고서입니다.

요청 시점에 별도 `ctest` 실패 로그는 제공되지 않았으므로, 현재 저장소 상태에서 빌드와 테스트를 직접 실행해 재현 여부를 확인했습니다.

## 3. 테스트 재현 결과

다음 명령으로 현재 빌드와 테스트를 확인했습니다.

```powershell
cmake --build build
ctest --test-dir build --output-on-failure
```

실행 결과는 다음과 같습니다.

- 빌드 성공
- 전체 20개 테스트 실행
- 20개 테스트 모두 통과
- 실패 테스트 0개

따라서 현재 코드 기준으로는 `EXPECT_EQ` 실패가 재현되지 않았습니다.

## 4. EXPECT_EQ 실패 기대/실제 차이 요약

현재 `ctest` 결과가 Green이므로 기대값과 실제값의 차이는 없습니다.

요약하면 다음과 같습니다.

- `SHealthBMITest` 전체 20개 테스트 통과
- `Underweight`, `Normal`, `Overweight`, `Obesity` 비율 검증 모두 통과
- `calculateBmi()` 반환 레코드 수 검증 모두 통과
- `getBmiRatio(ageClass, type)` 결과 검증 모두 통과

실패 로그가 별도로 제공되면 해당 로그의 `Expected equality of these values` 블록을 기준으로 기대값/실제값 차이를 재분석해야 합니다.

## 5. 결함 위치 분석

### 5.1 `SHealth::calculateBmi()`

위치:

- `src/main/cpp/SHealth.cpp:39`

현재 구현 흐름은 다음과 같습니다.

```cpp
int SHealth::calculateBmi(const std::string& filename) {
    resetState();

    auto loadedRecords = loadRecords(filename);
    if (!loadedRecords) {
        return 0;
    }

    records = std::move(*loadedRecords);
    imputeMissingWeights();
    calculateBmiValues();
    ratiosByAgeClass = aggregateRatios();

    return static_cast<int>(records.size());
}
```

확인 결과:

- 재계산 전 `resetState()`로 이전 상태를 초기화합니다.
- 파일 로드 실패 시 `0`을 반환합니다.
- 유효 레코드를 로드한 뒤 체중 보정, BMI 계산, 연령대별 비율 집계를 수행합니다.
- 현재 테스트 기준 결함은 재현되지 않았습니다.

### 5.2 `SHealth::getBmiRatio()`

위치:

- `src/main/cpp/SHealth.cpp:55`

현재 구현은 다음과 같습니다.

```cpp
double SHealth::getBmiRatio(int ageClass, int type) {
    const auto category = categoryFromType(type);
    const auto ratioIt = ratiosByAgeClass.find(ageClass);
    if (!category || ratioIt == ratiosByAgeClass.end()) {
        return 0.0;
    }

    return ratioIt->second.values[static_cast<std::size_t>(*category)];
}
```

확인 결과:

- 알 수 없는 BMI 타입은 `0.0`을 반환합니다.
- 존재하지 않는 연령대도 `0.0`을 반환합니다.
- 타입 코드 `100`, `200`, `300`, `400`은 `categoryFromType()`에서 BMI 범주로 매핑됩니다.
- 현재 테스트 기준 결함은 재현되지 않았습니다.

### 5.3 BMI 계산식

위치:

- `src/main/cpp/SHealth.cpp:199`

현재 구현은 다음과 같습니다.

```cpp
double SHealth::calculateBmiValue(double weight, double height) const {
    if (weight <= 0.0 || height <= 0.0) {
        return 0.0;
    }
    const double heightInMeters = height / CentimetersPerMeter;
    return weight / (heightInMeters * heightInMeters);
}
```

확인 결과:

- 키 단위는 cm로 입력받아 m로 변환합니다.
- BMI 공식 `weight / (heightInMeters * heightInMeters)`를 사용합니다.
- 체중 또는 키가 유효하지 않으면 `0.0`을 반환합니다.
- 현재 테스트 기준 결함은 재현되지 않았습니다.

### 5.4 BMI 분류 경계

위치:

- `src/main/cpp/StandardBmiClassifier.cpp:9`

현재 구현은 다음과 같습니다.

```cpp
BmiCategory StandardBmiClassifier::classify(double bmi) const {
    if (bmi <= UnderweightMax) {
        return BmiCategory::Underweight;
    }
    if (bmi < NormalMax) {
        return BmiCategory::Normal;
    }
    return bmi < OverweightMax ? BmiCategory::Overweight : BmiCategory::Obesity;
}
```

확인 결과:

- `18.5` 이하는 저체중입니다.
- `18.5` 초과, `23.0` 미만은 정상체중입니다.
- `23.0` 이상, `25.0` 미만은 과체중입니다.
- `25.0` 이상은 비만입니다.
- 현재 테스트 기준 경계값 결함은 재현되지 않았습니다.

## 6. 결함 심각도 분류

현재 재현 가능한 결함이 없으므로 심각도는 `Info`로 분류합니다.

근거:

- `ctest` 전체 20개 테스트가 통과했습니다.
- `EXPECT_EQ` 실패 로그가 제공되지 않았습니다.
- 현재 구현에서 BMI 계산, 타입 분류, 연령대 집계, 비율 조회의 실패가 관찰되지 않았습니다.
- 제품 기능이 중단되거나 잘못된 결과를 반환한다는 재현 증거가 없습니다.

단, 별도 환경에서 실패 로그가 존재한다면 다음 기준으로 재분류해야 합니다.

- `Critical`: 프로그램 크래시, 데이터 손상, 핵심 기능 전체 불능
- `Major`: BMI 계산 또는 비율 산출이 주요 시나리오에서 잘못됨
- `Minor`: 일부 경계값 또는 예외 입력에서만 잘못됨
- `Info`: 테스트/문서/환경 차이 또는 재현 불가

## 7. 최소 변경 수정 방안

현재 코드 기준으로는 수정이 필요하지 않습니다.

제안 diff:

```diff
# no changes required
```

실패 로그가 제공되어 실제 기대/실제 차이가 확인될 경우에는 다음 원칙으로 최소 수정합니다.

- `calculateBmi()` 흐름은 public API 동작을 유지하면서 결함 지점만 수정합니다.
- BMI 계산식은 `height / 100.0` 변환과 `weight / (heightInMeters * heightInMeters)` 공식을 유지합니다.
- BMI 분류 경계는 요구사항의 `18.5`, `23.0`, `25.0` 기준을 명확히 반영합니다.
- `getBmiRatio()`는 알 수 없는 연령대/타입에 대해 `0.0`을 반환하는 현재 방어 동작을 유지합니다.
- C++17 표준 라이브러리와 현재 코드 스타일을 따릅니다.

## 8. Green 확인 절차

Windows PowerShell 환경에서는 다음 명령을 사용합니다.

```powershell
cmake --build build
ctest --test-dir build --output-on-failure
```

빌드 성공 후 CTest 결과가 다음 상태이면 Green으로 판단합니다.

```text
100% tests passed, 0 tests failed out of 20
```

Unix 계열 셸 또는 `&&`를 지원하는 셸에서는 다음처럼 조건부 실행할 수 있습니다.

```sh
cmake --build build && ctest --test-dir build --output-on-failure
```

## 9. 결론

현재 저장소 상태에서는 BMI 테스트 실패가 재현되지 않았고, `SHealth::calculateBmi()`, `SHealth::getBmiRatio()`, BMI 계산식, BMI 분류 경계 모두 기존 테스트를 통과했습니다.

따라서 현재 조치 사항은 코드 수정이 아니라 실패 로그 확보입니다. 실패 로그가 제공되면 기대값/실제값 차이를 기준으로 결함 위치와 심각도를 다시 특정하고, 최소 diff 형태로 수정안을 제시해야 합니다.
