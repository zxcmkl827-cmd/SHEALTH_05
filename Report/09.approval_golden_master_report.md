# SHEALTH BMI Approval/Golden Master 테스트 구현 보고서

## 1. 작성 목적

본 문서는 SHEALTH BMI 프로그램의 콘솔 출력 기반 회귀 테스트를 Approval/Golden Master 방식으로 설계하고 구현한 내용을 정리한 보고서입니다.

요구사항은 C++17, Google Test, CMake 환경에서 `TexttestFixture` 출력을 기준으로 Golden Master를 구성하고, `ApprovalTests::Approvals::verify()` 메소드를 사용해 expected 출력 파일과 실제 실행 결과를 비교하는 것이었습니다.

## 2. 작업 대상

- 실행 대상: `src/main/cpp/SHealthBMI.cpp`
- Golden Master 테스트: `src/test/cpp/TexttestFixture.cpp`
- ApprovalTests Google Test main: `src/test/cpp/ApprovalTestsMain.cpp`
- 승인 파일: `src/test/cpp/approval_tests/TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.approved.txt`
- 빌드 설정: `CMakeLists.txt`
- CI 설정: `.github/workflows/ci.yml`
- 입력 데이터: `shealth.dat`

## 3. 설계 개요

Golden Master 테스트는 현재 프로그램의 사용자 관찰 가능 결과인 stdout을 회귀 기준으로 고정합니다.

`SHealthBMI` 실행 파일을 실제로 실행하고, `shealth.dat`를 읽어 출력되는 연령대별 BMI 분포 문자열을 ApprovalTests의 승인 파일과 비교합니다. 이 방식은 내부 구현 변경이 있더라도 최종 콘솔 출력이 유지되는지를 검증하므로 리팩토링 회귀 테스트에 적합합니다.

검증 흐름은 다음과 같습니다.

1. `SHealthGoldenMasterTest` 테스트 실행 파일이 시작됩니다.
2. `TexttestFixture` 테스트가 `SHealthBMI` 실행 파일을 소스 루트에서 실행합니다.
3. stdout을 문자열로 캡처합니다.
4. 줄 끝 차이를 줄이기 위해 CRLF를 LF로 정규화하고 trailing newline을 제거합니다.
5. `ApprovalTests::Approvals::verify(actual)`로 승인 파일과 비교합니다.

## 4. Expected 출력 파일 생성 및 보관 전략

ApprovalTests.cpp의 표준 파일 명명 규칙을 사용했습니다.

- 승인 파일: `*.approved.txt`
- 실패 시 생성 파일: `*.received.txt`

현재 기준 출력은 다음 파일에 보관합니다.

```text
src/test/cpp/approval_tests/TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.approved.txt
```

승인 파일은 테스트 소스와 가까운 `approval_tests` 하위 디렉터리에 두었습니다. 이는 ApprovalTests의 `useApprovalsSubdirectory("approval_tests")` 설정과 맞으며, 테스트 파일 주변에 생성되는 `received`/`approved` 파일을 한 곳에 모아 관리할 수 있습니다.

출력 변경이 의도된 경우에는 테스트 실패 시 생성되는 `.received.txt`를 검토한 뒤 `.approved.txt`로 교체합니다.

```powershell
copy src\test\cpp\approval_tests\TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.received.txt src\test\cpp\approval_tests\TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.approved.txt
```

출력 변경이 의도되지 않은 경우에는 `.approved.txt`를 수정하지 않고 원인 코드를 수정해야 합니다.

## 5. Google Test 구현 내용

`src/test/cpp/ApprovalTestsMain.cpp`는 ApprovalTests가 Google Test와 연동될 수 있도록 전용 main을 제공합니다.

```cpp
#define APPROVALS_GOOGLETEST
#include "ApprovalTests.hpp"

auto approvalDirectory =
    ApprovalTests::Approvals::useApprovalsSubdirectory("approval_tests");
```

`src/test/cpp/TexttestFixture.cpp`는 실제 실행 파일의 stdout을 캡처한 뒤 ApprovalTests로 검증합니다.

핵심 검증 코드는 다음과 같습니다.

```cpp
TEST(TexttestFixture, SHealthBmiOutputMatchesGoldenMaster) {
    const auto dataFile = std::filesystem::path(SHEALTH_GOLDEN_DATA_FILE);
    const auto executable = std::filesystem::path(SHEALTH_GOLDEN_EXECUTABLE_FILE);
    const auto workingDir = std::filesystem::path(SHEALTH_GOLDEN_WORKING_DIR);

    ASSERT_TRUE(std::filesystem::exists(dataFile)) << dataFile.string();
    ASSERT_TRUE(std::filesystem::exists(executable)) << executable.string();
    ASSERT_TRUE(std::filesystem::exists(workingDir)) << workingDir.string();

    const std::string actual =
        trimTrailingNewlines(normalizeLineEndings(runSHealthBmi(executable, workingDir)));
    ApprovalTests::Approvals::verify(actual);
}
```

기존 수동 파일 비교 방식의 `EXPECT_EQ(expected, actual)` 대신 `ApprovalTests::Approvals::verify(actual)`를 사용하도록 변경했습니다. 따라서 승인 파일 생성, received 파일 생성, diff 비교 흐름은 ApprovalTests가 담당합니다.

## 6. CMake/ctest 통합

`CMakeLists.txt`에는 ApprovalTests.cpp 의존성을 `FetchContent`로 추가했습니다.

```cmake
FetchContent_Declare(
    ApprovalTests
    GIT_REPOSITORY https://github.com/approvals/ApprovalTests.cpp.git
    GIT_TAG v.10.13.0
)
FetchContent_MakeAvailable(ApprovalTests)
```

Golden Master 전용 테스트 실행 파일은 다음과 같이 구성했습니다.

```cmake
add_executable(SHealthGoldenMasterTest
    src/test/cpp/ApprovalTestsMain.cpp
    src/test/cpp/TexttestFixture.cpp
)
target_link_libraries(SHealthGoldenMasterTest PRIVATE
    shealth_lib
    ApprovalTests::ApprovalTests
    GTest::gtest
)
```

`gtest_main`을 링크하지 않고 `GTest::gtest`만 링크한 이유는 ApprovalTests가 Google Test 테스트 이름을 안정적으로 얻기 위해 자체 Google Test main 설정을 사용해야 하기 때문입니다.

테스트 실행 파일에는 다음 compile definition을 전달합니다.

- `SHEALTH_GOLDEN_DATA_FILE`: `shealth.dat` 절대 경로
- `SHEALTH_GOLDEN_EXECUTABLE_FILE`: `SHealthBMI` 실행 파일 경로
- `SHEALTH_GOLDEN_WORKING_DIR`: 실행 기준 디렉터리

마지막으로 `gtest_discover_tests(SHealthGoldenMasterTest)`를 통해 `ctest`에 자동 등록했습니다.

## 7. CI 자동 실행 구성

`.github/workflows/ci.yml`을 추가해 push와 pull request에서 자동으로 빌드와 테스트가 실행되도록 구성했습니다.

```yaml
name: CI

on:
  push:
  pull_request:

jobs:
  test:
    runs-on: windows-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Configure
        run: cmake -S . -B build

      - name: Build
        run: cmake --build build --config Release

      - name: Test
        run: ctest --test-dir build -C Release --output-on-failure
```

CI에서는 승인 파일이 저장소에 포함되어 있어야 하며, 출력이 승인 파일과 다르면 테스트가 실패합니다. 이 실패는 의도하지 않은 출력 회귀를 빠르게 확인하기 위한 신호입니다.

## 8. 실행 방법

로컬 실행 절차는 다음과 같습니다.

```powershell
cmake -S . -B build
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

Golden Master 테스트만 실행하려면 다음 명령을 사용할 수 있습니다.

```powershell
ctest --test-dir build -C Debug -R TexttestFixture --output-on-failure
```

직접 테스트 실행 파일을 실행할 수도 있습니다.

```powershell
.\build\Debug\SHealthGoldenMasterTest.exe
```

## 9. 검증 결과

구현 후 다음 명령으로 전체 테스트를 실행했습니다.

```powershell
cmake --build build --config Debug
ctest --test-dir build -C Debug --output-on-failure
```

실행 결과는 다음과 같습니다.

- 전체 21개 테스트 실행
- 기존 BMI 단위 테스트 20개 통과
- Approval/Golden Master 테스트 1개 통과
- 실패 테스트 0개

최종 결과는 `100% tests passed, 0 tests failed out of 21`입니다.

## 10. 기대 효과

이번 Golden Master 테스트를 통해 다음 효과를 얻을 수 있습니다.

- `SHealthBMI` 콘솔 출력 포맷과 계산 결과 회귀 감지
- 리팩토링 시 내부 구조 변경과 외부 출력 변경을 분리해 검증
- 수동 expected 파일 비교 대신 ApprovalTests 표준 승인 흐름 사용
- CMake, ctest, GitHub Actions CI에서 동일한 테스트 경로 제공
- 출력 변경이 발생했을 때 `.received.txt`와 `.approved.txt` 비교를 통한 리뷰 가능

## 11. 주의 사항

Approval/Golden Master 테스트는 현재 동작을 기준으로 고정하는 회귀 테스트입니다. 따라서 현재 출력이 요구사항상 올바른지 확인한 뒤 승인 파일을 갱신해야 합니다.

또한 승인 파일 갱신은 단순히 테스트를 통과시키기 위한 작업이 아니라, 출력 변경이 의도된 변경인지 검토하는 승인 절차로 다루어야 합니다.
