# SHEALTH BMI 기능 개선 테스트 계획 보고서

## 1. 작성 목적

본 문서는 `SHealth` BMI 모듈의 기능 개선 항목에 대해 시니어 QA 리드 관점에서 수립한 테스트 계획을 보고서 형태로 요약합니다.

상세 테스트 계획서는 `docs/feature_test_plan.md`에 작성했으며, 본 보고서는 테스트 범위, 우선순위, 경계값, 예외/특이 케이스, 커버리지 목표와 측정 전략을 중심으로 정리합니다.

## 3. 테스트 계획 수립 배경

현재 `SHealth` 구현은 CSV 파일 로딩, 행 파싱, 체중 `0` 평균 보정, BMI 계산, 연령대별 BMI 범주 비율 산출을 수행합니다.

기능 개선 요구사항은 기존 기능을 유지하면서 다음 동작을 추가 검증해야 합니다.

| 개선 항목 | QA 핵심 검증 포인트 |
|---|---|
| 특정 연령대 BMI 분포 비율 | 연령대 경계값, 범주별 비율, 분모 0 처리 |
| `height == 0` 평균치 보정 | 같은 연령대 평균 키 보정, 보정 불가 데이터 제외 |
| BMI 정상 범위 사용자 목록 조회 | `18.5 < bmi < 23.0`, ID 보존, 입력 순서 유지 |
| 전체 사용자 대비 BMI 범주 비율 | 전체 유효 BMI 사용자 기준 분모, 합계 100% 근사 |
| SRP 기반 리팩토링 | 기존 public 동작 보존, 상태 초기화, 회귀 방지 |

## 4. 테스트 설계 원칙

테스트는 C++17, Google Test, CMake 환경을 기준으로 설계했습니다. 기본 구조는 `TEST_F` 기반 fixture를 사용하며, 테스트 fixture에서 임시 CSV 파일을 생성해 공개 API로 관찰 가능한 동작을 검증합니다.

주요 설계 원칙은 다음과 같습니다.

- 구현 세부 private 함수보다 `calculateBmi`, `getBmiRatio`, 추가 조회 API의 public 동작을 우선 검증합니다.
- BMI 경계값처럼 독립성이 높은 정책은 `StandardBmiClassifierTest`로 분리합니다.
- 비율과 BMI 계산처럼 부동소수점 오차가 발생할 수 있는 값은 `EXPECT_NEAR`를 사용합니다.
- 기능 추가 전후의 회귀 방지를 위해 기존 체중 보정, 연령대별 비율, 반복 호출 상태 초기화를 P0로 둡니다.
- CSV 파싱 오류, 보정 불가, 파일 없음, 분모 0은 정상 경로와 같은 수준으로 테스트합니다.

## 5. TEST_F 기반 단위 테스트 우선순위

### 5.1 P0: 핵심 기능과 회귀 방지

P0는 기능 개선 완료 여부를 판단하는 필수 테스트입니다.

| ID | 테스트명 예시 | 검증 내용 |
|---|---|---|
| FT-P0-001 | `CalculateBmi_ReturnsLoadedRecordCount_ForValidCsv` | 정상 CSV 처리 건수 반환 |
| FT-P0-002 | `GetBmiRatio_ReturnsAgeClassDistribution_ForMixedCategories` | 연령대별 BMI 범주 비율 계산 |
| FT-P0-003 | `CalculateBmi_ImputesZeroWeight_WithSameAgeClassAverage` | 체중 `0` 평균 보정 |
| FT-P0-004 | `CalculateBmi_ImputesZeroHeight_WithSameAgeClassAverage` | 키 `0` 평균 보정 |
| FT-P0-005 | `GetNormalBmiUsers_ReturnsOnlyUsersBetweenNormalBoundaries` | 정상 BMI 사용자 목록 필터링 |
| FT-P0-006 | `GetOverallBmiDistribution_ReturnsRatios_ForAllValidUsers` | 전체 BMI 범주 비율 계산 |
| FT-P0-007 | `CalculateBmi_ResetsPreviousState_OnRepeatedCalls` | 반복 호출 시 상태 초기화 |

### 5.2 P1: 경계값과 입력 안정성

P1은 결함 발생 가능성이 높은 경계값과 malformed input을 검증합니다.

| 영역 | 주요 케이스 |
|---|---|
| BMI 경계 | `18.5`, `18.5 초과`, `23.0`, `25.0` |
| 나이 경계 | `19`, `20`, `29`, `30`, `70`, `79`, `80` |
| 파일 오류 | 존재하지 않는 파일 입력 |
| CSV 오류 | 컬럼 부족, 비숫자 나이/체중/키, 빈 필드 |
| 유효성 오류 | 음수 체중, 음수 키 |

### 5.3 P2: 정책 확인과 특이 케이스

P2는 정책 명확화와 장기 유지보수 안정성을 위한 테스트입니다.

| 영역 | 주요 케이스 |
|---|---|
| 빈 데이터 | 빈 파일, 헤더만 있는 파일 |
| 빈 줄 | CSV 중간 빈 줄 무시 후 계속 처리 |
| 보정 불가 | 같은 연령대의 모든 체중 또는 키가 `0` |
| 조회 정책 | 정상 BMI 사용자 목록의 CSV ID 보존과 입력 순서 유지 |
| 잘못된 조회 | 지원하지 않는 ageClass 또는 BMI type code |

## 7. 예외 및 특이 케이스

예외와 특이 케이스는 시스템 안정성 관점에서 별도 테스트 대상으로 분류했습니다.

| 케이스 | 기대 동작 |
|---|---|
| 파일 없음 | `calculateBmi`가 `0` 반환, 내부 상태 초기화 |
| 빈 파일 | 예외 없이 0건 처리 |
| 헤더만 존재 | 모든 비율 `0.0` |
| 컬럼 부족 | 해당 행 스킵 |
| 추가 컬럼 | 필수 4개 컬럼만 사용하거나 정책 확정 |
| 비숫자 필드 | 해당 행 스킵 |
| 음수 체중/키 | 해당 행 스킵 |
| BMI `18.5`, `23.0` | 정상 BMI 사용자 목록에서 제외 |
| 같은 객체 반복 호출 | 이전 결과가 다음 결과에 남지 않음 |

## 8. 커버리지 목표

기능 개선 완료 기준 커버리지 목표는 다음과 같이 설정했습니다.

| 대상 | 목표 |
|---|---:|
| `src/main/cpp/SHealth.cpp` 라인 커버리지 | 90% 이상 |
| `src/main/cpp/SHealth.cpp` 브랜치 커버리지 | 85% 이상 |
| `StandardBmiClassifier.cpp` 라인 커버리지 | 95% 이상 |
| 신규 기능 코드 라인 커버리지 | 90% 이상 |
| 전체 `src/main/cpp` 라인 커버리지 | 90% 이상 |

커버리지 산정 범위는 `shealth_lib`에 포함되는 비즈니스 로직을 중심으로 하며, `main` 진입점, 외부 라이브러리, 테스트 코드, 빌드 산출물은 제외합니다.

## 9. gcov/lcov 측정 전략

GCC 또는 MinGW 기반 환경에서는 별도 커버리지 빌드 디렉터리에서 측정합니다.

```bash
cmake -S . -B build-coverage -DCMAKE_BUILD_TYPE=Debug \
  -DCMAKE_CXX_FLAGS="--coverage -O0 -g" \
  -DCMAKE_EXE_LINKER_FLAGS="--coverage"
cmake --build build-coverage
ctest --test-dir build-coverage --output-on-failure
lcov --capture --directory build-coverage --output-file build-coverage/coverage.info
lcov --remove build-coverage/coverage.info "*/_deps/*" "*/src/test/*" "*/SHealthBMI.cpp" --output-file build-coverage/coverage.filtered.info
genhtml build-coverage/coverage.filtered.info --output-directory build-coverage/coverage-html
```

Windows 로컬 환경에서 `lcov` 사용이 어렵다면 WSL 또는 Linux CI에서 동일 소스 기준 커버리지 잡을 운영하는 것을 권장합니다.

## 10. 커버리지 개선 전략

1. P0 테스트로 정상 흐름, 체중/키 보정, 정상 BMI 사용자 조회, 전체 분포 계산을 먼저 덮습니다.
2. P1 테스트로 BMI 경계, 나이 경계, malformed CSV, 파일 open 실패, 음수 값 스킵 경로를 보강합니다.
3. P2 테스트로 분모 0, 보정 불가, 잘못된 타입 코드, 빈 데이터 같은 방어 분기를 검증합니다.
4. private 함수 직접 테스트가 필요할 정도로 커버리지 사각지대가 크면 `CsvHealthRecordReader`, `MissingValueImputer`, `BmiStatisticsService` 등으로 책임을 분리합니다.
5. 신규 API가 추가될 때마다 정상 케이스, 빈 결과 케이스, 경계값 케이스를 최소 1개씩 함께 추가합니다.

## 11. 완료 기준

기능 개선 테스트 계획의 완료 기준은 다음과 같습니다.

- `docs/feature_test_plan.md`가 작성되어 테스트 범위와 우선순위가 명확히 정의되어 있습니다.
- P0 테스트 항목이 모두 구현되고 통과해야 합니다.
- P1 중 BMI 경계값, 나이 경계값, malformed CSV 테스트가 통과해야 합니다.
- 기능 개선 코드의 라인 커버리지가 90% 이상이어야 합니다.
- `ctest --test-dir build --output-on-failure`가 성공해야 합니다.
- 커버리지 리포트에서 미검증 라인이 남으면 위험도와 제외 사유를 별도 기록해야 합니다.

## 12. 결론

이번 테스트 계획은 기능 개선 구현 전에 QA 기준을 먼저 고정하기 위한 산출물입니다. 핵심은 기존 연령대별 BMI 비율과 체중 보정 동작을 회귀 테스트로 보호하면서, 키 보정, 정상 BMI 사용자 목록, 전체 BMI 분포라는 신규 기능을 P0 테스트로 끌어올리는 것입니다.

커버리지는 단순 라인 수치보다 분기와 정책 검증을 함께 관리해야 합니다. 특히 파일 오류, malformed CSV, 보정 불가, 경계값 분류는 실제 결함 가능성이 높은 영역이므로 커버리지 개선의 우선 대상으로 관리하는 것이 적절합니다.
