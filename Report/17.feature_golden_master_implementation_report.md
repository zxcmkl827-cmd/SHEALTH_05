# SHEALTH BMI 기능 Golden Master 회귀 테스트 구현 보고서

## 1. 작성 목적

본 문서는 추가된 SHEALTH BMI 기능에 대해 출력 기반 Golden Master 회귀 테스트를 설계하고 구현한 내용을 정리한 보고서입니다.

요구사항은 C++17, Google Test, CMake 환경에서 기존 `src/test/cpp/TexttestFixture.cpp`, `src/test/cpp/ApprovalTestsMain.cpp`를 활용해 다음 항목을 구현하는 것이었습니다.

- expected 출력 파일 생성 및 보관 전략
- Google Test에서 파일 비교 테스트 구현
- CMake/CTest 통합
- CI 자동 실행 구성
- 테스트 코드, CMake 수정안, 실행 방법 정리

## 2. 작업 대상

- Golden Master 테스트 진입점: `src/test/cpp/ApprovalTestsMain.cpp`
- Golden Master 테스트 구현: `src/test/cpp/TexttestFixture.cpp`
- 기존 승인 파일: `src/test/cpp/approval_tests/TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.approved.txt`
- 신규 expected 파일: `src/test/cpp/golden_master/SHealthFeatureSnapshot.expected.txt`
- 빌드 및 테스트 설정: `CMakeLists.txt`
- CI 설정: `.github/workflows/ci.yml`
- 주요 검증 대상:
  - `src/main/cpp/SHealth.h`
  - `src/main/cpp/SHealth.cpp`
  - `src/main/cpp/SHealthBMI.cpp`

## 3. 구현 전 상태

기존 프로젝트에는 ApprovalTests.cpp 기반 Golden Master 테스트가 이미 존재했습니다.

- `ApprovalTestsMain.cpp`에서 `APPROVALS_GOOGLETEST`를 정의하고 ApprovalTests와 Google Test를 연결
- `Approvals::useApprovalsSubdirectory("approval_tests")`로 승인 파일 보관 위치 지정
- `TexttestFixture.SHealthBmiOutputMatchesGoldenMaster` 테스트에서 `SHealthBMI` 실행 파일의 표준 출력을 캡처해 approved 파일과 비교

다만 기존 Golden Master는 CLI 출력만 검증했기 때문에, 기능 개선 항목인 키 `0` 보정, 정상 BMI 사용자 목록, 전체 BMI 범주 비율을 한 번에 관찰하는 출력 기반 회귀 테스트는 부족했습니다.

## 4. Expected 출력 파일 생성 및 보관 전략

Golden Master 산출물은 두 계층으로 관리하도록 구성했습니다.

### 4.1 기존 CLI 출력 승인 파일

기존 ApprovalTests 방식은 유지했습니다.

- 보관 위치: `src/test/cpp/approval_tests`
- 승인 파일: `TexttestFixture.SHealthBmiOutputMatchesGoldenMaster.approved.txt`
- 역할: 실제 `SHealthBMI` 실행 파일의 사용자 출력이 변경되지 않았는지 검증

이 파일은 ApprovalTests.cpp 관례에 따라 actual 출력이 approved 파일과 다르면 received 파일을 생성하고 테스트를 실패시킵니다.

### 4.2 신규 기능 Snapshot expected 파일

추가 기능은 별도 expected 파일로 보관했습니다.

- 보관 위치: `src/test/cpp/golden_master`
- expected 파일: `SHealthFeatureSnapshot.expected.txt`
- 실패 시 생성 파일: `SHealthFeatureSnapshot.received.txt`

신규 Snapshot은 테스트 코드가 deterministic CSV 입력을 임시 파일로 생성한 뒤, `SHealth` public API 결과를 텍스트 출력으로 직렬화해 expected 파일과 비교합니다.

expected 파일에 고정한 출력 범위는 다음과 같습니다.

- 처리된 레코드 수
- 20대부터 70대까지 연령대별 저체중/정상/과체중/비만 비율
- 전체 사용자 기준 저체중/정상/과체중/비만 비율
- 정상 BMI 사용자 목록
- 키 `0` 평균 보정 후 결과
- 연령대별 통계에서는 제외되지만 전체 통계에는 포함되는 나이 `80` 케이스

## 5. Google Test 파일 비교 테스트 구현

`TexttestFixture.cpp`에 신규 테스트 `SHealthFeatureSnapshotMatchesExpectedFile`를 추가했습니다.

테스트 흐름은 다음과 같습니다.

1. `writeFeatureGoldenMasterInput()`에서 임시 CSV 입력 생성
2. `SHealth::calculateBmi()` 실행
3. `getBmiRatio()`, `getOverallBmiRatio()`, `getNormalBmiUsers()` 결과를 문자열로 직렬화
4. `SHealthFeatureSnapshot.expected.txt`를 읽어 actual 출력과 비교
5. 비교 실패 시 `SHealthFeatureSnapshot.received.txt`에 actual 출력 저장
6. expected와 actual이 같으면 이전 실패에서 남은 received 파일 제거

출력은 환경 차이에 흔들리지 않도록 다음 정책을 적용했습니다.

- `std::fixed`, `std::setprecision(6)`로 부동소수점 자릿수 고정
- CRLF/LF 차이를 제거하기 위해 줄바꿈 정규화
- 테스트 데이터는 외부 입력 파일에 의존하지 않고 테스트 내부에서 생성
- expected 파일 경로는 CMake compile definition으로 주입

## 6. CMake/CTest 통합

`CMakeLists.txt`에는 Golden Master 테스트 실행에 필요한 설정을 추가했습니다.

주요 변경 사항은 다음과 같습니다.

- `SHEALTH_GOLDEN_EXPECTED_DIR` compile definition 추가
- `SHealthGoldenMasterTest`가 `SHealthBMI` 실행 파일 빌드 이후 실행되도록 `add_dependencies()` 추가
- `gtest_discover_tests()`에 CTest label 적용
- Golden Master 테스트의 working directory를 프로젝트 루트로 고정

CTest 라벨은 다음과 같이 분리했습니다.

- `unit`: `SHealthBMITest`
- `golden-master`: `SHealthGoldenMasterTest`

따라서 전체 테스트는 기존처럼 실행할 수 있고, Golden Master만 별도로 실행할 수도 있습니다.

```powershell
ctest --test-dir build -C Release --output-on-failure
ctest --test-dir build -C Release -L golden-master --output-on-failure
```

## 7. CI 자동 실행 구성

`.github/workflows/ci.yml`은 기존 Windows CI 흐름을 유지하면서 Golden Master 실패 분석이 가능하도록 보강했습니다.

CI 실행 단계는 다음과 같습니다.

1. `actions/checkout@v4`
2. `cmake -S . -B build`
3. `cmake --build build --config Release`
4. `ctest --test-dir build -C Release --output-on-failure`
5. 실패 시 received 파일 artifact 업로드

업로드 대상 received 파일은 다음과 같습니다.

- `src/test/cpp/approval_tests/*.received.txt`
- `src/test/cpp/golden_master/*.received.txt`

이를 통해 CI에서 Golden Master가 실패하면 변경된 actual 출력을 artifact로 내려받아 expected 파일 갱신 여부를 검토할 수 있습니다.

## 8. 실행 방법

Windows PowerShell 기준 실행 방법은 다음과 같습니다.

```powershell
cmake -S . -B build
cmake --build build --config Release
ctest --test-dir build -C Release --output-on-failure
```

PowerShell 환경에 따라 `&&` 체인이 동작하지 않을 수 있으므로, 조건부 실행이 필요하면 다음처럼 `$LASTEXITCODE`를 확인하는 방식이 안전합니다.

```powershell
cmake -S . -B build
if ($LASTEXITCODE -ne 0) { exit $LASTEXITCODE }
cmake --build build --config Release
if ($LASTEXITCODE -ne 0) { exit $LASTEXITCODE }
ctest --test-dir build -C Release --output-on-failure
```

Golden Master 테스트만 실행하려면 다음 명령을 사용합니다.

```powershell
ctest --test-dir build -C Release -L golden-master --output-on-failure
```

구현 후 다음 명령으로 빌드와 테스트를 검증했습니다.

cmake --build build --config Release
ctest --test-dir build -C Release --output-on-failure

## 11. 결론

이번 작업으로 기존 CLI 출력 Golden Master는 유지하면서, 추가된 기능의 핵심 public 동작을 파일 기반 Snapshot으로 고정했습니다.

단위 테스트가 세부 조건을 좁게 검증하고, Golden Master 테스트가 여러 기능이 함께 만들어내는 최종 출력을 넓게 감시하는 구조가 되었습니다. CI에서도 동일한 ctest 명령으로 자동 실행되며, 실패 시 received 파일을 artifact로 남겨 승인 파일 갱신 여부를 검토할 수 있습니다.
