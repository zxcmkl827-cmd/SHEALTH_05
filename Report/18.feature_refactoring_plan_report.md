# SHealth.cpp 모던 C++ 리팩토링 계획 보고서

## 1. 작성 목적

본 문서는 `src/main/cpp/SHealth.cpp`에 정의된 메소드들을 대상으로, 테스트 Green 상태를 유지하면서 진행할 수 있는 모던 C++ 리팩토링 계획을 정리한 보고서입니다.

리팩토링 목표는 다음과 같습니다.

- 조건 분기 축소와 중복 제거
- 커밋 단위로 쪼갠 점진적 개선
- 타입/정책 분리
- 전략 패턴, 테이블 기반 구조, 함수 분해 적용
- 매직 넘버 상수화와 C++17 스타일 개선
- 매 단계 `cmake --build`와 `ctest` 기반 검증

## 3. 현재 구현 요약

현재 `SHealth`는 BMI 계산 기능의 facade 역할을 수행하면서도 여러 책임을 동시에 가지고 있습니다.

- CSV 파일 읽기
- CSV row 파싱
- 체중과 키 결측치 보정
- BMI 값 계산
- BMI 분류 전략 호출
- 연령대별 BMI 비율 집계
- 전체 BMI 비율 집계
- 정상 BMI 사용자 목록 조회

다만 이전 개선을 통해 일부 기반은 이미 마련되어 있습니다.

- `BmiClassifier`와 `StandardBmiClassifier`가 분리되어 BMI 분류 전략은 교체 가능한 구조입니다.
- `SHealth::BmiUser`는 `id`, `age`, `weight`, `height`, `bmi`를 보존합니다.
- CSV 컬럼 인덱스, BMI type code, 연령대 범위 등 일부 매직 넘버는 익명 namespace 상수로 분리되어 있습니다.
- `calculateBmi()`는 `load -> impute -> calculate -> aggregate` 흐름을 비교적 명확하게 유지하고 있습니다.

## 4. 주요 개선 지점

### 4.1 BMI 타입 코드 변환

현재 `categoryFromType()`은 `100`, `200`, `300`, `400` 외부 type code를 `switch`로 `BmiCategory`에 매핑합니다.

개선 방향은 다음과 같습니다.

- `constexpr std::array<std::pair<int, BmiCategory>, 4>` 기반 매핑 테이블 도입
- `switch` 제거
- CLI 출력과 ratio 조회가 같은 type code 정책을 공유하도록 확장 가능성 확보
- public API의 `int type`은 유지하되 내부 변환만 테이블 기반으로 격리

### 4.2 연령대 정책 중복

현재 연령대 정책은 `MinAgeClass`, `MaxAgeClass`, `AgeClassStep`, `AgeClasses`, `toAgeClass()`에 나뉘어 있습니다.

개선 방향은 다음과 같습니다.

- `AgeClassPolicy`로 연령대 계산과 지원 연령대 목록을 한 곳에 모음
- `20~79` 범위와 `10`년 단위 정책을 단일 소스로 관리
- 연령대별 통계와 결측치 평균 계산이 같은 정책을 사용하도록 정리

### 4.3 체중/키 결측치 보정 중복

현재 `calculateAverageWeights()`, `calculateAverageHeights()`, `imputeMissingWeights()`, `imputeMissingHeights()`는 거의 같은 구조를 공유합니다.

개선 방향은 다음과 같습니다.

- 값 getter와 setter를 받는 공통 평균 계산/보정 helper 도입
- `weight == 0.0`, `height == 0.0` 결측 정책을 명명된 함수로 분리
- C++17 람다 또는 작은 템플릿 helper로 중복 제거
- 우선 `SHealth.cpp` 내부 helper로 시작하고, 안정화 후 `MissingValueImputer`로 파일 분리

### 4.4 연령대별/전체 비율 집계 중복

현재 `calculateRatioSet(int ageClass)`와 `calculateOverallRatioSet()`은 비슷한 카운트와 퍼센트 변환 흐름을 갖습니다.

개선 방향은 다음과 같습니다.

- predicate를 받는 공통 집계 함수 도입
- `countByCategory()`와 `toRatioSet()` 함수 분리
- BMI 유효성 판단을 `hasValidBmi()`로 명명
- 연령대별 통계와 전체 통계의 분모 차이는 테스트로 고정

### 4.5 CSV 파싱 책임

현재 `split()`, `loadRecords()`, `parseRecord()`가 `SHealth` private 메소드로 존재합니다.

개선 방향은 다음과 같습니다.

- 먼저 `SHealth.cpp` 내부 `CsvSchema`, `CsvHealthRecordParser` 형태로 책임을 분리
- 이후 안정화되면 `CsvHealthRecordReader` 파일로 이동
- `catch (...)`는 `std::invalid_argument`, `std::out_of_range` 중심으로 좁힘
- 파일 없음, malformed row skip, 음수 값 skip 정책은 기존 테스트 기준으로 유지

## 5. 단계별 리팩토링 계획

모든 단계는 테스트 Green 상태에서만 다음 단계로 넘어갑니다.

공통 검증 명령은 다음과 같습니다.

```bash
cmake --build build
ctest --test-dir build --output-on-failure
```

### 5.1 0단계: Baseline Green 확인

체크리스트:

- 현재 테스트 전체가 Green인지 확인
- 실패가 있으면 리팩토링 전에 원인 분리
- 기존 공개 동작을 변경하지 않는다는 기준 확정

검증 포인트:

- 존재하지 않는 파일은 `0` 반환
- 유효하지 않은 BMI type은 `0.0` 반환
- 연령대별 통계는 `20~79`만 포함
- 전체 통계는 유효 BMI면 연령대 밖도 포함

- enum index 변환을 `toCategoryIndex()` 같은 함수로 집중

검증 방법:

## 7. 리스크와 대응 방안

| 리스크 | 영향 | 대응 |
|---|---|---|
| 연령대 정책 변경 중 전체 통계 기준이 함께 바뀜 | `19`, `80` 사용자의 전체 비율 계산이 깨질 수 있음 | 연령대별 통계와 전체 통계의 predicate를 분리하고 기존 테스트로 고정 |
| BMI type code를 enum으로 직접 바꾸며 public API 변경 | 기존 호출부와 테스트가 깨질 수 있음 | 기존 `int` API는 유지하고 내부 adapter만 테이블화 |
| 결측치 보정 공통화 중 weight와 height 정책이 섞임 | 보정 불가 레코드의 BMI 포함 여부가 달라질 수 있음 | getter/setter와 결측 판단 함수를 명확히 분리 |
| CSV 파서 분리 중 row skip 정책 변경 | record count와 정상 사용자 목록 결과가 달라질 수 있음 | malformed row, 음수 값, ID 보존 테스트를 매 단계 확인 |
| CLI 반복 제거 중 출력 포맷 변경 | Golden Master 테스트 실패 | 출력 문자열 변경은 별도 의사결정으로 분리 |

## 8. 결론

`SHealth.cpp`의 리팩토링은 큰 구조 변경을 한 번에 수행하기보다, 현재 테스트를 안전망으로 삼아 작은 커밋으로 진행하는 것이 적합합니다.

가장 안전한 순서는 BMI type 매핑 테이블화, 연령대 정책 통합, 결측치 보정 중복 제거, 비율 집계 중복 제거, CSV 파싱 분리, 서비스 파일 분리, CLI 반복 제거입니다.

이 순서는 public API와 기존 테스트를 유지하면서 조건 분기와 중복을 줄이고, 이후 `SHealth`를 facade에 가까운 구조로 축소할 수 있는 기반을 제공합니다.
