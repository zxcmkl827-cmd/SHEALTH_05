✅ TDD 기반 암호 검사기 또는 BMI 프로젝트용 프롬프트
💬 [1] 테스트 코드부터 작성하고 싶은 경우 (TDD 시작용)

요구사항 기반으로 먼저 실패하는 Google Test 테스트 코드를 작성해줘.
각 테스트는 단일 조건을 검사하며, 테스트 이름은 명확하게 지어줘.
조건: [암호는 길이 8자 이상 / 숫자 포함 / 대문자 포함 여부를 검사하고 규칙 충족 수에 따라 STRONG/NORMAL/WEAK/INVALID로 구분]

또는 BMI 프로젝트라면:

BMI 계산기 프로그램에 대해 TDD를 시작하고 싶어.
1. BMI 계산 로직에 대한 테스트
2. 평균 보정(weight=0 or height=0) 테스트
3. BMI 카테고리 분류 테스트
4. 연령대별 BMI 통계 계산 기능 테스트

이 4가지에 대해 먼저 실패하는 테스트 코드부터 작성해줘.

💬 [2] 테스트 통과를 위한 최소 구현 코드 요청

위에서 작성한 테스트 코드가 통과할 수 있도록 최소한의 C++ 구현 코드를 작성해줘.
조건에 맞게 `PasswordStrengthMeter` 혹은 `BMICalculator` 클래스를 만들어서 테스트를 통과시키는 데 집중해줘.

💬 [3] 클린코드 원칙을 적용한 리팩토링 요청

현재 작성된 C++ 코드를 클린코드 원칙(SRP, DRY, 의미 있는 네이밍, 함수 분리 등)에 따라 리팩토링해줘.
특히 다음에 주의해줘:
- Magic Number 제거
- 한 함수의 책임 최소화
- 테스트하기 쉽도록 분리

💬 [4] 설계 다이어그램이 필요한 경우

현재 PasswordStrengthMeter 또는 SHealth BMI 프로그램을 클래스 다이어그램 형태로 표현해줘.
- 주요 클래스, 책임, 메서드 포함
- 테스트 코드와 도메인 로직의 관계도 함께 보여줘

💬 [5] 테스트 커버리지를 점검하고 싶은 경우

PasswordStrengthMeter의 현재 테스트 코드가 모든 분기 조건을 커버하고 있는지 점검해줘.
- 빠진 경계 조건이나 예외 상황이 있다면 테스트 케이스를 추가로 제안해줘.
