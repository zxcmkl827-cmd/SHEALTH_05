# SHealth BMI 기능개선 요구사항 분석

## 1. 목적과 범위

본 문서는 `README.md`의 `4. 기능 개선` 항목을 C++17 구현 관점에서 재정리한다.

대상 프로젝트는 CMake, Google Test 기반의 SHEALTH BMI 계산 모듈이며, 개선 범위는 다음 공개 동작을 중심으로 한다.

1. SRP에 따른 책임 분리 등 리팩토링
2. 특정 연령대의 BMI 분포 비율 계산 기능
3. `height == 0`인 경우에 대한 평균치 보정 로직
4. BMI 정상 범위 사용자 목록 조회 기능
5. 전체 사용자 대비 각 BMI 범주 비율 계산 기능
6. Google Test 기준 테스트 시나리오 목록

## 2. 기능개선 요구사항 요약

| 번호 | 개선 항목 | C++ 구현 관점 | 주요 산출물 | QA 확인 포인트 |
|---:|---|---|---|---|
| 0 | SRP에 따른 책임 분리 등 리팩토링 | CSV 로딩, 레코드 파싱, 누락값 보정, BMI 계산, BMI 분류, 통계 집계를 분리한다. | `HealthRecord`, `CsvHealthRecordReader`, `MissingValueImputer`, `BmiCalculator`, `BmiStatisticsService` 등 역할 기반 구조 | 기존 `calculateBmi`, `getBmiRatio` 동작 보존, 상태 초기화, 예외 입력 처리 |
| 1 | 특정 연령대의 BMI 분포 비율 계산 | 10년 단위 연령대별로 저체중/정상/과체중/비만 비율을 계산한다. | `getBmiRatio(ageClass, type)` 또는 `getAgeClassDistribution(ageClass)` | 20, 29, 30, 79, 80 경계값, 데이터 0건 분모 처리 |
| 2 | `height == 0` 평균치 보정 | 같은 연령대의 유효 키 평균으로 `height == 0` 값을 보정한다. | `calculateAverageHeights`, `imputeMissingHeights` | 모든 키가 0인 연령대, 음수 키, 보정 후 BMI 재계산 |
| 3 | BMI 정상 범위 사용자 목록 조회 | BMI가 `18.5 < bmi < 23.0`인 사용자 레코드 목록을 조회한다. | `getNormalBmiUsers()` 또는 `findUsersByBmiCategory(BmiCategory::Normal)` | ID 보존 필요성, 정렬 기준, 경계값 18.5/23.0 제외 |
| 4 | 전체 사용자 대비 각 BMI 범주 비율 계산 | 연령대와 무관하게 전체 유효 사용자 기준 BMI 범주 비율을 산출한다. | `getOverallBmiDistribution()` | 연령대 제외 사용자 포함 여부, 보정 불가 레코드 제외 여부, 합계 100% 근사 |
| 5 | Google Test 시나리오 목록 | 기능별 정상/경계/예외/회귀 테스트를 번호로 관리한다. | `TEST_F` 기반 테스트 케이스 목록 | Given-When-Then 구조, `EXPECT_NEAR` 기반 비율 검증 |

## 3. 구현 책임 분리 방안

| 책임 | 현재 구현 관점 | 개선 방향 | 테스트 단위 |
|---|---|---|---|
| 데이터 모델 | `SHealth::HealthRecord`가 private nested struct로 존재한다. ID는 저장하지 않는다. | 정상 BMI 사용자 목록 조회를 위해 `id`, `age`, `weight`, `height`, `bmi`를 가진 독립 모델로 분리한다. | 레코드 생성, ID 보존, 값 유효성 |
| CSV 읽기 | `SHealth::loadRecords`, `parseRecord`, `split`가 파일 처리와 파싱을 담당한다. | `CsvHealthRecordReader`로 분리해 파일 open 실패, 빈 줄, 컬럼 부족, 비숫자 값을 한 곳에서 처리한다. | 파싱 성공/실패, malformed row 스킵 |
| 누락값 보정 | `imputeMissingWeights`가 체중 0만 보정한다. | `MissingValueImputer`에서 체중 0과 키 0을 각각 같은 연령대 평균으로 보정한다. | 체중 보정, 키 보정, 보정 불가 케이스 |
| BMI 계산 | `calculateBmiValue`가 `SHealth` 내부에 있다. | 순수 함수 또는 `BmiCalculator`로 분리해 단위 테스트가 쉽게 수행되도록 한다. | 키 cm to m 변환, 0/음수 입력 |
| BMI 분류 | `BmiClassifier`, `StandardBmiClassifier`로 이미 분리되어 있다. | 현재 구조를 유지하고 경계값 테스트를 강화한다. | 18.5, 23.0, 25.0 경계 |
| 통계 집계 | `aggregateRatios`, `calculateRatioSet`가 연령대별 비율을 계산한다. | `BmiStatisticsService`로 분리해 연령대별 분포와 전체 분포를 모두 제공한다. | 연령대별/전체 비율, 분모 0 |
| 조회 API | 현재는 `getBmiRatio(ageClass, type)`만 제공한다. | 정상 BMI 사용자 목록, 전체 분포 조회 API를 추가한다. | 정상 범위 목록, 전체 범주 비율 |

## 4. 기능별 상세 요구사항

### 4.1 특정 연령대의 BMI 분포 비율 계산

| 항목 | 요구사항 |
|---|---|
| 입력 | `ageClass`는 `20, 30, 40, 50, 60, 70` 중 하나를 사용한다. `type`은 저체중 `100`, 정상 `200`, 과체중 `300`, 비만 `400`을 사용하거나 enum 기반 API로 대체한다. |
| 처리 | 해당 연령대의 유효 BMI 레코드를 BMI 범주별로 집계하고 `범주 건수 / 해당 연령대 유효 건수 * 100`으로 계산한다. |
| 출력 | 비율은 `double`로 반환한다. 대상 데이터가 없거나 입력 코드가 유효하지 않으면 `0.0`을 반환한다. |
| 구현 주의 | 매직 넘버는 `enum class BmiCategory`와 변환 함수로 격리한다. 비율 비교 테스트는 부동소수점 오차를 고려해 `EXPECT_NEAR`를 사용한다. |

### 4.2 `height == 0` 평균치 보정

| 항목 | 요구사항 |
|---|---|
| 입력 | CSV의 키 값이 `0`이면 누락값으로 간주한다. 음수 키는 유효하지 않은 데이터로 처리한다. |
| 처리 | 같은 10년 단위 연령대의 `height > 0` 표본 평균을 계산한 뒤 `height == 0` 레코드에 대입한다. |
| 출력 | 보정된 키로 BMI를 계산한다. 보정 가능한 평균이 없으면 해당 레코드는 BMI 통계에서 제외한다. |
| 구현 주의 | 기존 체중 0 보정과 동일한 정책을 사용하되, 평균 계산 시 보정 대상 값은 분모에서 제외한다. BMI 계산은 키 보정 이후 한 번만 수행한다. |

### 4.3 BMI 정상 범위 사용자 목록 조회

| 항목 | 요구사항 |
|---|---|
| 입력 | `calculateBmi(filename)`로 로딩과 보정, BMI 계산이 완료된 상태를 기준으로 한다. |
| 처리 | BMI가 `18.5 < bmi < 23.0`인 사용자만 필터링한다. `18.5`는 저체중, `23.0`은 과체중이므로 제외한다. |
| 출력 | 사용자 ID를 포함한 레코드 목록을 반환한다. 최소 반환 정보는 `id`, `age`, `weight`, `height`, `bmi`이다. |
| 구현 주의 | 현재 `HealthRecord`는 ID를 저장하지 않으므로 CSV 파싱 단계에서 ID 보존이 필요하다. 반환 목록의 정렬 기준은 입력 순서 유지를 기본값으로 한다. |

### 4.4 전체 사용자 대비 각 BMI 범주 비율 계산

| 항목 | 요구사항 |
|---|---|
| 입력 | 전체 유효 레코드를 대상으로 한다. 연령대가 20~79 범위를 벗어나도 BMI 계산이 유효하면 전체 통계에는 포함할지 정책을 명확히 한다. |
| 처리 | 저체중/정상/과체중/비만 건수를 전체 유효 BMI 건수로 나누어 비율을 계산한다. |
| 출력 | `BmiCategory`별 비율을 담은 `RatioSet` 또는 `std::map<BmiCategory, double>`을 반환한다. |
| 구현 주의 | 연령대별 통계와 전체 통계의 분모가 다를 수 있으므로 API 이름과 테스트명에 기준 집합을 명확히 드러낸다. |

## 5. Google Test 기준 테스트 시나리오 목록

1. `TC-FR-001`: 정상 CSV를 읽으면 레코드 수를 반환하고 기존 연령대별 BMI 비율이 계산된다.
2. `TC-FR-002`: BMI `18.5`, `18.6`, `22.9`, `23.0`, `24.9`, `25.0`이 기준에 맞게 분류된다.
3. `TC-FR-003`: 나이 `20`, `29`, `30`, `39`, `70`, `79`가 올바른 10년 단위 연령대로 집계된다.
4. `TC-FR-004`: 나이 `19`와 `80`은 연령대별 통계에서 제외된다.
5. `TC-FR-005`: 특정 연령대에 저체중/정상/과체중/비만이 섞여 있으면 각 범주 비율이 정확히 계산된다.
6. `TC-FR-006`: 특정 연령대에 유효 BMI 데이터가 없으면 모든 범주 비율이 `0.0`이다.
7. `TC-FR-007`: 체중 `0`은 같은 연령대의 0이 아닌 체중 평균으로 보정된다.
8. `TC-FR-008`: 같은 연령대의 모든 체중이 `0`이면 보정 불가 레코드는 BMI 통계에서 제외된다.
9. `TC-FR-009`: 키 `0`은 같은 연령대의 0이 아닌 키 평균으로 보정된다.
10. `TC-FR-010`: 같은 연령대의 모든 키가 `0`이면 보정 불가 레코드는 BMI 통계에서 제외된다.
11. `TC-FR-011`: 음수 체중 또는 음수 키가 포함된 행은 안전하게 스킵된다.
12. `TC-FR-012`: 컬럼 부족, 빈 숫자 필드, 비숫자 필드가 포함된 행은 예외 없이 스킵된다.
13. `TC-FR-013`: 정상 BMI 사용자 목록 조회 시 `18.5 < bmi < 23.0` 사용자만 반환된다.
14. `TC-FR-014`: 정상 BMI 사용자 목록은 CSV의 사용자 ID를 보존한다.
15. `TC-FR-015`: 정상 BMI 사용자 목록은 입력 순서를 유지한다.
16. `TC-FR-016`: 전체 사용자 대비 BMI 범주 비율이 전체 유효 BMI 건수를 분모로 계산된다.
17. `TC-FR-017`: 전체 BMI 범주 비율과 특정 연령대 BMI 범주 비율이 서로 다른 분모를 사용함을 검증한다.
18. `TC-FR-018`: 전체 유효 BMI 데이터가 없으면 전체 범주 비율이 모두 `0.0`이다.
19. `TC-FR-019`: 같은 `SHealth` 객체로 `calculateBmi`를 반복 호출해도 이전 통계와 사용자 목록이 남지 않는다.
20. `TC-FR-020`: 존재하지 않는 파일을 입력하면 `0`을 반환하고 모든 조회 API가 빈 결과 또는 `0.0`을 반환한다.

## 6. 구현 우선순위

1. `HealthRecord`에 ID를 포함하고 CSV 파싱 결과를 명확한 데이터 모델로 정리한다.
2. 기존 체중 보정, BMI 계산, 연령대별 비율 테스트를 먼저 Green 상태로 고정한다.
3. `height == 0` 평균 보정 로직과 테스트를 추가한다.
4. 정상 BMI 사용자 목록 조회 API와 테스트를 추가한다.
5. 전체 BMI 범주 비율 API와 테스트를 추가한다.
6. 마지막으로 `SHealth` 내부 책임을 reader, imputer, statistics service 등으로 점진 분리한다.

## 7. 결론

1. 기능 개선의 핵심은 통계 API 추가보다 먼저 데이터 모델과 책임 경계를 명확히 하는 것이다.
2. `height == 0` 보정은 체중 0 보정과 동일한 연령대 평균 정책을 적용하되, 보정 불가 레코드의 통계 포함 여부를 테스트로 고정해야 한다.
3. 정상 BMI 사용자 목록 조회를 위해서는 CSV의 ID를 반드시 보존해야 하며, 이는 현재 구현 대비 데이터 모델 변경이 필요한 지점이다.
4. 전체 BMI 범주 비율은 기존 연령대별 비율과 분모가 다르므로 별도 API와 별도 테스트로 관리해야 한다.
